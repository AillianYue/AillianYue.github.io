C++基础：STL

发表于2025-07-03|更新于2025-10-01|C++C++基础
|总字数:5.3k|阅读时长:20分钟|浏览量:

VERSION：3.0.0

STL概述

STL（Standard Template Library）是C++标准库的核心组成部分，通过模板化设计实现了数据结构和算法的通用化，大幅提升代码复用性和效率。其核心思想是将数据存储（容器）、数据访问（迭代器）、数据处理逻辑（函数对象）、通用操作（算法） 解耦，使各组件可灵活组合。

STL四大核心组件

组件 核心作用 头文件（核心）
容器 Container 管理一组元素的存储，是数据的“容器” 按容器类型（如<vector>、<set>）
迭代器 Iterator 泛化的指针，作为容器和算法的“桥梁”，统一访问容器元素的接口 <iterator>
函数对象 Function Object 行为类似函数的对象（重载()运算符），作为算法的“逻辑参数” <functional>
算法 Algorithm 通用的操作逻辑（排序、查找、变换等），通过迭代器操作容器，函数对象定制逻辑 <algorithm>
  • 迭代器是算法与容器的桥梁：算法不直接操作容器，而是通过迭代器访问/修改容器元素；
  • 函数对象是算法的“逻辑扩展”：算法的核心逻辑（如比较、运算）由函数对象定义，而非硬编码在算法内；
  • 组件解耦带来灵活性：新增容器无需修改算法，新增算法可适配所有兼容迭代器的容器。

容器（Container）

容器是容纳一组元素的对象，按元素组织方式和访问特性分类，所有容器均提供通用基础接口，且元素需满足“可赋值（Assignable）”（公有拷贝构造+=赋值）。

容器通用功能

操作/类型 作用
默认构造函数 构造空容器
关系运算符（==/!=/<等） 按字典序比较容器元素
begin()/end() 获取指向容器首元素/尾后位置的迭代器
clear() 清空容器所有元素
empty() 判断容器是否为空（效率高于size()==0）
size() 返回容器元素个数（forward_list不支持）
swap(s1, s2) 交换两个同类型容器的内容（高效，仅交换内部指针）
S::iterator 可读写迭代器类型
S::const_iterator 只读迭代器类型

容器分类及详解

顺序容器

（元素线性排列，按插入顺序存储）

容器类型 核心特性 优势场景 劣势场景 核心接口补充
array<T, N> 固定大小的数组封装，替代原生数组，更安全 随机访问快，内存连续 大小不可变，不支持插入/删除 operator[]（随机访问）、data()（获取原生指针）
vector<T> 动态可扩展数组，内存连续 随机访问快，尾部插入/删除快 中间/头部插入/删除慢（需移动元素）、扩容有开销 capacity()（容量）、reserve(n)（预分配容量）、push_back()/pop_back()
deque<T> 双端队列，分段连续内存 两端插入/删除快，随机访问较快 中间插入/删除慢，随机访问比vector慢 push_front()/pop_front()、operator[]
list<T> 双向链表，非连续内存 任意位置插入/删除快 不支持随机访问，遍历慢 splice()（接合链表）、push_front()/push_back()
forward_list 单向链表，仅支持单向遍历 内存开销最小，任意位置插入/删除快 不支持随机访问，无size()，操作需针对“元素后位置” insert_after()/erase_after()、emplace_after()
  • vector的“容量（capacity）”：实际分配的内存大小，reserve(n)可预分配容量避免频繁扩容；
    • resize(n)：改变容器的实际大小（size）
      • 如果 n > 当前size：新增的元素会被初始化（默认值 0，或指定值）
      • 如果 n < 当前size：截断容器，删除超出 n 的元素
      • 容量（capacity）可能会自动增长（按需分配），但不会减少
    • reserve(n)：仅预留容量（capacity），不改变实际大小（size）和元素内容
      • 如果 n > 当前capacity：分配至少能容纳 n 个元素的内存，capacity 变为≥n
      • 如果 n ≤ 当前capacity：无任何效果
      • 不会创建 / 删除任何元素，size 和元素内容不变
  • list的splice()：s1.splice(p, s2, q1, q2)将s2的[q1, q2)区间元素移动到s1的p位置前（无拷贝，仅修改指针）。

关联容器

（按键排序存储，底层多为红黑树）

容器类型 核心特性 适用场景
set<T> 单重集合，键=元素，键唯一，自动排序 去重+有序存储，快速查找
multiset<T> 多重集合，键=元素，键可重复，自动排序 允许重复元素的有序存储
map<K, V> 单重映射，键值对（pair<K,V>），键唯一，按键排序 键值映射，快速按键查找
multimap<K,V> 多重映射，键值对，键可重复，按键排序 一个键对应多个值的映射

核心接口：

  • insert()：插入元素（map插入pair<K,V>或make_pair(k,v)）；
  • find(key)：查找键为key的元素，返回迭代器（未找到返回end()）；
  • count(key)：统计键为key的元素个数（set/map返回0或1，multiset/multimap返回实际个数）；
  • lower_bound(key)/upper_bound(key)：返回键≥key/键>key的第一个元素迭代器。

无序关联容器

（C++11，按哈希存储，底层为哈希表）

容器类型 核心特性 对比关联容器
unordered_set<T> 无序单重集合，键唯一，哈希存储 查找/插入/删除平均O(1)，无排序
unordered_multiset<T> 无序多重集合，键可重复，哈希存储 同上，允许重复键
unordered_map<K,V> 无序单重映射，键唯一，哈希存储 同上，键值对
unordered_multimap<K,V> 无序多重映射，键可重复，哈希存储 同上，允许重复键
  • 自定义类型作为键时，需提供哈希函数（重载hash模板）和==运算符；
  • 平均时间复杂度O(1)，最坏O(n)（哈希冲突）。

容器适配器

（封装顺序容器实现特定数据结构）

适配器不直接暴露迭代器，仅提供特定操作接口，底层依赖顺序容器（可指定）。

适配器类型 底层默认容器 核心特性（访问规则） 核心接口
stack<T> deque<T> 后进先出（LIFO） push()/pop()/top()
queue<T> deque<T> 先进先出（FIFO） push()/pop()/front()/back()
priority_queue<T> vector<T> 优先级最高的元素先出（默认大顶堆） push()/pop()/top()
  • priority_queue的底层容器需支持随机访问（vector/deque）；可通过自定义函数对象修改优先级（如小顶堆：priority_queue<T, vector<T>, greater<T>>）；
  • 适配器无迭代器，不支持随机访问。

迭代器（Iterator）

迭代器是“泛化的指针”，统一了不同容器的元素访问方式，算法通过迭代器操作容器，无需关心容器类型。

迭代器声明

  1. 普通迭代器（可读写）

核心语法：容器类型::iterator 迭代器名;适用于需要读取并修改vector元素的场景。

  1. 常量迭代器（只读，不可修改）

核心语法：容器类型::const_iterator 迭代器名;适用于仅读取元素、不修改的场景（比如遍历const vector，或想避免误修改元素）。

  1. 反向迭代器（从后往前遍历）

核心语法：

  • 普通反向迭代器：容器类型::reverse_iterator 迭代器名;
  • 常量反向迭代器：容器类型::const_reverse_iterator 迭代器名;适用于需要从vector末尾向前遍历的场景。
  1. C++11 及以上：auto 简化声明（推荐）

无需手动写冗长的迭代器类型，编译器会自动推导迭代器的具体类型，代码更简洁易维护。

迭代器区间

STL算法的输入通常是“左闭右开”区间[first, last)：

  • 包含first指向的元素，不包含last指向的元素；
  • 合法区间：first经n次++（n>0）后等于last。

迭代器辅助函数

函数 作用
advance(it, n) 将迭代器it移动n步（n可负，仅双向/随机访问迭代器支持）
distance(it1, it2) 计算it1到it2的距离（随机访问迭代器O(1)，其他O(n)）
begin(c)/end(c) C++11，获取容器c的首/尾迭代器（支持const版本）

迭代器失效

  • vector：扩容时所有迭代器失效；中间插入/删除，该位置后的迭代器失效；
  • deque：两端插入/删除仅end()失效；中间插入/删除，所有迭代器失效；
  • list/forward_list：插入/删除仅指向被删元素的迭代器失效；
  • 关联/无序关联容器：插入不失效，删除仅指向被删元素的迭代器失效。

函数对象（Function Object）

函数对象是重载了 () 运算符的类 / 结构体的实例，它能像普通函数一样被调用，但比普通函数更灵活：

  • 可携带状态（成员变量）；
  • 可被 STL 适配器（如 bind、not1）适配；
  • 编译器更容易内联优化，性能通常优于普通函数指针。

STL 提供了大量现成的函数对象（如 greater<int>、plus<int>），也支持自定义函数对象，以下是最常用的场景和用法。

常用场景

场景 1：配合 STL 算法进行操作

这是函数对象最核心的场景。STL 算法（如 sort、transform、find_if）通常需要传入 “操作逻辑”（比如排序规则、过滤条件），函数对象是天然的选择。

用标准库自带函数对象

STL 在 <functional> 头文件中提供了常用函数对象，分为三类：

  • 算术类：plus<T>（加法）、minus<T>（减法）、multiplies<T>（乘法）等；
  • 关系类：greater<T>（大于）、less<T>（小于）、equal_to<T>（等于）等；
  • 逻辑类：logical_and<T>（与）、logical_or<T>（或）等。

自定义函数对象（作为过滤条件）

如果标准库函数对象不满足需求，可自定义函数对象作为算法的谓词（判断条件）。

示例：筛选偶数

场景 2：作为容器的比较规则

STL 关联容器（map、set）默认用 less<T> 排序，可通过函数对象自定义排序规则。

示例：map 按 key 的字符串长度排序

场景 3：带状态的函数对象

普通函数 / 匿名 lambda（无捕获）无法携带状态，而函数对象可通过成员变量保存状态，适合需要 “累计 / 记录” 的场景。

示例：统计 vector 中偶数的个数

函数对象 vs lambda 表达式

C++11 后的 lambda 表达式可替代大部分简单函数对象，但函数对象仍有优势：

  • 需复用逻辑时：函数对象可定义一次，多处使用；
  • 需携带状态且状态复杂时：函数对象的成员变量更清晰；
  • 需适配 STL 适配器时：函数对象（尤其是标准库自带的）兼容性更好。

算法（Algorithm）

STL 算法集中在 <algorithm>、<numeric>（数值计算）等头文件中，所有算法都通过迭代器操作容器元素，不依赖具体容器类型（vector/string/list 等都适用），分为两大核心类型：

  • 非修改型算法：仅读取元素（如查找、统计），不改变容器内容；
  • 修改型算法：会修改元素值或调整元素顺序（如排序、替换）。

下面按 “使用场景 + 核心算法 + 代码示例” 的方式，讲解最常用的 STL 算法。

遍历与查找

遍历容器元素、查找特定值 / 满足条件的元素、统计元素个数。

算法 功能 示例场景
for_each 遍历所有元素并执行自定义操作 打印元素、批量修改（非返回）
find 查找第一个匹配指定值的元素 找值为 5 的元素
find_if 查找第一个满足自定义条件的元素 找第一个偶数
count 统计指定值出现的次数 统计 3 出现的次数
count_if 统计满足条件的元素个数 统计偶数个数

排序与排列

对容器元素排序、反转顺序、随机打乱、取 TopN 元素。

算法 功能 示例场景
sort 快速排序（不稳定），默认升序 升序 / 降序排序
stable_sort 稳定排序（保留相等元素相对顺序） 排序后保留原始顺序
reverse 反转元素顺序 倒序输出
shuffle 随机打乱元素（C++11+） 随机排序数组
nth_element 找第 n 大 / 小元素，且左边≤它、右边≥它 找第 3 大元素

修改与变换

批量修改元素、替换指定值、删除元素（逻辑删除）、复制元素到新容器。

算法 功能 示例场景
transform 批量变换元素（返回新值） 所有元素 + 5、字符串转大写
replace 替换所有指定值为新值 把 3 替换为 9
remove 逻辑删除指定值（返回新尾迭代器） 删除所有 3（需 erase 配合）
copy 复制元素到另一个容器 复制偶数到新 vector

数值计算

求和、求内积、计算前缀和、相邻元素差值。

算法 功能 示例场景
accumulate 累加元素（默认求和，可自定义操作） 求和、求乘积
inner_product 两个容器内积（对应元素相乘再求和） 向量内积计算
partial_sum 计算前缀和 累加前缀数组
