C++基础：字符串

std::string 基础

C 风格字符串与 std::string 对比

C 风格字符串的局限

C 风格字符串本质是char数组 + 末尾\0终止符，核心痛点：

  • 内存手动管理：需手动malloc/free/new/delete，易内存泄漏 / 重复释放；
  • 安全风险：strcpy/strcat无边界检查，易缓冲区溢出；
  • 接口简陋：仅靠strlen/strcmp等函数，无直接拼接 / 替换功能；
  • 长度计算低效：strlen需遍历到\0，时间复杂度 O (n)；
  • 语法不友好：不支持运算符重载（如==/+=）。

std::string 的优势

std::string 是 C++ 标准库封装的字符串类，完美解决 C 风格痛点：

  • 自动内存管理：底层自动扩容 / 释放，无需手动操作；
  • 接口丰富：内置拼接、查找、替换、转换等数十个成员函数；
  • 安全访问：at()方法带越界检查，抛异常而非直接崩溃；
  • 语法友好：支持+=/==/<等运算符重载；
  • 兼容 STL：可直接使用迭代器、STL 算法；
  • 内存优化：内置小字符串优化（SSO），减少堆分配。

std::string 基础操作

初始化与赋值

方式 示例 说明
直接初始化 string s1;
string s2("hello");
string s3{"hello"};		 默认构造 / 字符串字面量初始化
拷贝初始化 string s3 = s2;
string s4(s2);		 深拷贝，独立内存
移动赋值（C++11+） string s5 = std::move(s2); 转移所有权，s2 变为空串
子串初始化 string s6("hello world", 5); 取前 5 个字符：”hello”

长度与容量

主要接口：

  • size()/length()：等价，返回字符串实际字符数（不含 \0）；
  • capacity()：返回当前分配的内存可容纳的字符数（含预留空间）；
  • reserve(n)：预分配至少 n 个字符的内存（仅扩容，不改变实际长度）；
  • resize(n)：调整字符串长度为 n，不足补 \0 / 截断多余字符；
  • shrink_to_fit()：释放未使用的预留内存（C++11+，非强制）。

元素访问

方法 说明 安全性
s[index] 随机访问，无越界检查 越界导致未定义行为
s.at(index) 随机访问，有越界检查 越界抛out_of_range
s.front() 访问第一个字符（等价于 s [0]） 空串访问未定义行为
s.back() 访问最后一个字符（等价于 s [s.size ()-1]） 空串访问未定义行为
s.data() 返回char*（C++11 后同 c_str ()） 只读（修改未定义）
s.c_str() 返回const char*（带 \0 终止符） 只读

空串判断与清空

  • s.empty()：判断是否为空串（size ()==0），效率高（O (1)）；
  • s.clear()：清空字符串（size () 变为 0，capacity 不变）。

std::string 修改操作

拼接

方法 示例 说明
+= s += "world" 最常用，支持字符串 / 字符
append() s.append("world") 支持子串、重复拼接
push_back(c) s.push_back('!') 追加单个字符
emplace_back(c) s.emplace_back('!') C++11+，直接构造字符（更高效）

插入 / 删除 / 替换

  • 插入：s.insert(pos, str) 在 pos 位置插入字符串；
  • 删除：s.erase(pos, len) 从 pos 位置删除 len 个字符（len 默认删到末尾）；
  • 替换：s.replace(pos, len, new_str) 替换 pos 开始的 len 个字符为 new_str。

std::string 查找与比较

查找

核心查找函数（均返回size_t，未找到返回string::npos）：

  • s.find(str, pos)：从 pos 开始正向查找 str（默认 pos=0）；
  • s.rfind(str)：反向查找最后一次出现的 str；
  • s.find_first_of(str)：查找 str 中任意字符首次出现的位置；
  • s.find_last_of(str)：查找 str 中任意字符最后一次出现的位置；
  • s.find_first_not_of(str)：查找首个不在 str 中的字符位置。

比较

  • 运算符重载：==/!=/</>/<=/>=（按 ASCII 码逐字符比较）；
  • s.compare(str)：返回 int，0 = 相等，<0=s 更小，>0=s 更大（支持子串比较）。

std::string 数值转换

字符串转数值

函数 功能 示例
stoi 转 int stoi("123") → 123
stol 转 long stol("123456")
stod 转 double stod("3.14") → 3.14
stof 转 float stof("2.718")

注意：转换失败（如非数字）会抛invalid_argument或out_of_range异常。

数值转字符串

to_string()：支持 int/long/float/double 等类型，返回 std::string。

C 风格字符串与 std::string 互转

方向 方法 注意事项
std::string → C 风格 s.c_str()；s.data() 返回 const char*，只读
C 风格 → std::string 直接赋值 / 构造 自动处理 \0 终止符

std::string 内存优化与高级特性

小字符串优化（SSO）

  • 原理：std::string 对象内部预留一小段栈内存（通常 15-22 字节，依编译器），存储短字符串时无需分配堆内存；
  • 优势：短字符串（如 < 16 字节）避免堆分配开销，提升性能；
  • 注意：超过 SSO 阈值后，才会向堆申请内存。

emplace 系列方法（C++11+）

  • emplace_back(c)：直接在字符串末尾构造字符，避免临时对象（比 push_back 更高效）；
  • emplace(pos, c)：在 pos 位置构造字符（替代 insert 单个字符）。

移动语义应用（std::move）

  • 场景：当字符串不再使用时，用std::move转移所有权，避免深拷贝；
  • 效果：原字符串变为空，新字符串接管内存（O (1) 操作）。

特殊字符串类型与 Unicode 处理

std::wstring（wchar_t）

  • 基于wchar_t的宽字符字符串，用于存储多字节字符（如中文 / 日文）；
  • 底层是wchar_t数组，长度计算用wcslen，拼接 / 查找等接口与 std::string 一致。

std::u8/u16/u32string（C++20）

类型 字符类型 编码 用途
std::u8string char8_t UTF-8 通用 Unicode 编码
std::u16string char16_t UTF-16 Windows/Java 常用
std::u32string char32_t UTF-32 固定 4 字节 / 字符

std::string 健壮性保障

字符串迭代器

std::string 支持 STL 迭代器，遍历更灵活：

  • iterator：可读写迭代器；
  • const_iterator：只读迭代器（const 对象使用）；
  • reverse_iterator：反向迭代器；
  • cbegin()/cend()：返回 const_iterator（C++11+）。

STL 算法适配

std::string 可直接使用 STL 算法，提升代码复用性。

std::transform（批量修改）

批量修改字符（如转大小写、替换字符）：

std::find/std::replace（查找 / 替换）

  • std::find：查找单个字符（比 string::find 更通用）；
  • std::replace：批量替换字符。

异常安全与处理

异常场景

  • at()越界：抛std::out_of_range；
  • 数值转换失败：抛std::invalid_argument/std::out_of_range；
  • 内存分配失败：抛std::bad_alloc（极少发生）。

try-catch 捕获异常

性能优化

预分配容量（reserve）

  • 问题：频繁拼接字符串时，std::string 会自动扩容（每次扩容通常翻倍），导致多次内存分配 + 拷贝；
  • 优化：提前用reserve(n)预分配足够容量，避免频繁扩容。

避免隐式转换（const std::string & 传参）

  • 问题：传值（std::string s）会触发深拷贝，开销大；
  • 优化：函数参数用const std::string&，避免拷贝，且防止隐式转换（如 char * 转 std::string）。

大字符串拼接优化

  • 场景：拼接大量字符串（如日志、SQL 语句）；
  • 优化方案：
    1. 先用reserve预分配容量；
    2. 或使用std::stringstream（适合极大量拼接）。

性能优化总结

  1. std::string 核心优势是自动内存管理和丰富接口，彻底解决 C 风格字符串的安全与易用性问题；
  2. 性能优化核心是减少内存分配（reserve 预分配）、避免不必要拷贝（const & 传参、move 语义）；
  3. 健壮性保障依赖越界检查（at ()）、异常捕获和STL 算法适配。
