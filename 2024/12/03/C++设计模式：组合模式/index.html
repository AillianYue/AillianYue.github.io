C++设计模式：组合模式

发表于2024-12-03|更新于2025-12-22|C++C++设计模式
|总字数:825|阅读时长:3分钟|浏览量:

VERSION：1.0.0

组合模式（Composite Pattern）是一种结构型设计模式，核心思想是将对象组合成树形结构，以表示 “部分 - 整体” 的层次关系，使得客户端对单个对象（叶子节点）和组合对象（容器节点）的使用具有一致性。

简单来说，组合模式让 “单个元素” 和 “元素集合” 拥有相同的接口，客户端无需区分它们，可以统一处理。

组合模式的结构

组合模式包含 3 个核心角色：

  1. 抽象组件（Component）
    声明叶子节点和容器节点的共同接口，定义了对 “部分” 和 “整体” 的统一操作（如添加、删除子组件、获取信息等）。
  2. 叶子节点（Leaf）
    表示树形结构中的基本元素（不可再分割的 “部分”），没有子节点。实现抽象组件的接口，但通常不支持 “添加 / 删除子组件” 等操作（可空实现或抛出异常）。
  3. 容器节点（Composite）
    表示树形结构中的 “整体”，可以包含叶子节点或其他容器节点（形成嵌套结构）。实现抽象组件的接口，负责管理子组件（添加、删除、遍历），并在执行业务逻辑时递归处理子组件。

适用场景

  • 需要表示 “部分 - 整体” 层次结构（如文件系统、UI 组件树、组织结构树）。
  • 希望客户端统一处理单个对象和组合对象，无需区分类型。

例子

以 “文件系统” 为例，文件夹（Folder）可以包含文件（File）或其他文件夹，文件是不可再分割的叶子节点。我们用组合模式实现这一结构。

组合模式的优缺点

  • 优点：
    1. 客户端代码统一，无需区分 “部分” 和 “整体”。
    2. 符合开闭原则，新增叶子或容器节点无需修改现有代码。
    3. 便于构建复杂的树形结构，且操作简单。
  • 缺点：
    1. 若叶子和容器差异过大，强行统一接口可能导致部分方法在叶子节点中无意义（需特殊处理，如空实现或异常）。
    2. 设计较复杂，需要合理抽象共同接口。
