C++设计模式：工厂模式

发表于2024-12-03|更新于2025-12-22|C++C++设计模式
|总字数:1.8k|阅读时长:7分钟|浏览量:

VERSION：1.0.0

工厂模式是一种常用的创建型设计模式，其核心思想是将对象的创建过程封装起来，使客户端无需直接依赖具体产品类，而是通过工厂类间接获取对象实例。这种模式能有效降低代码耦合度，提高系统的可扩展性和可维护性。

工厂模式的三种主要类型

根据复杂度和适用场景，工厂模式可分为以下三种：简单工厂模式、工厂方法模式、抽象工厂模式。

简单工厂模式（Simple Factory）

简单工厂模式通过一个单一工厂类，根据输入参数创建不同类型的产品实例。
结构：

  • 抽象产品（Product）：定义所有产品的公共接口
  • 具体产品（ConcreteProduct）：实现抽象产品接口的具体类
  • 工厂（Factory）：根据参数创建对应具体产品的实例

示例：假设需要创建圆形、矩形等图形，客户端只需指定类型即可获取对应图形对象。

优缺点：

  • 优点：实现简单，客户端无需知道具体产品类名，只需关注类型参数
  • 缺点：新增产品时需修改工厂类（违反 “开闭原则”），适合产品类型固定且较少的场景

工厂方法模式（Factory Method）

工厂方法模式将对象创建延迟到具体工厂子类，每个具体产品对应一个具体工厂。
结构：

  • 抽象产品（Product）：定义产品接口
  • 具体产品（ConcreteProduct）：实现产品接口
  • 抽象工厂（Factory）：定义创建产品的接口（纯虚函数）
  • 具体工厂（ConcreteFactory）：实现抽象工厂接口，创建对应具体产品

示例：图形绘制工具（工厂方法版）
为每种图形创建专属工厂，新增图形时只需添加新的产品和工厂类。

优缺点：

  • 优点：符合 “开闭原则”（新增产品只需新增产品类和工厂类，无需修改现有代码）
  • 缺点：系统复杂度增加（每增加一个产品需对应增加一个工厂）

抽象工厂模式（Abstract Factory）

抽象工厂模式用于创建一系列相互关联或依赖的产品族（如不同风格的 UI 组件：按钮 + 文本框）。
结构：

  • 抽象产品族（多个抽象产品）：定义一系列相关产品的接口
  • 具体产品族（多个具体产品）：实现抽象产品接口，属于同一产品族
  • 抽象工厂：定义创建产品族中所有产品的接口
  • 具体工厂：实现抽象工厂接口，创建对应产品族的所有产品

示例：跨平台 UI 组件
假设需要创建 Windows 和 Mac 风格的按钮和文本框，每种风格构成一个产品族。

优缺点：

  • 优点：能保证同一产品族的产品相互匹配，便于切换产品族
  • 缺点：新增产品族中的产品时，需修改抽象工厂和所有具体工厂（违反 “开闭原则”）

三种工厂模式的对比

模式类型 核心特点 适用场景
简单工厂模式 单一工厂类根据参数创建产品 产品类型少且固定，不常扩展
工厂方法模式 每个产品对应一个工厂子类 产品类型可能扩展，需遵循开闭原则
抽象工厂模式 创建一系列相关的产品族 需管理多个产品族，且产品族内产品相互依赖
