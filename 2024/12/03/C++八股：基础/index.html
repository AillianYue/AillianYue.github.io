C++中如何正确比较浮点数是否相等？

正确比较方法：

  • 绝对误差比较：适用于数值范围已知的情况
  • 相对误差比较：适用于数值范围变化较大的情况
  • ULP比较：基于浮点数表示的精度的比较方法
  • 结合绝对和相对误差：综合两种方法的优势

浮点数在计算机中采用IEEE 754标准表示，存在以下问题：

  • 精度限制：浮点数有有限的精度位数，无法精确表示所有实数
  • 舍入误差：运算过程中会产生舍入误差，累积后可能显著
  • 表示误差：某些十进制数在二进制中无法精确表示（如0.1）
  • 不同运算顺序可能产生不同结果

浮点数表示基础 IEEE 754单精度浮点数结构：

1位符号位

  • 8位指数位
  • 23位尾数位

常见浮点数陷阱

  • 累积误差：多次运算后误差累积
  • 吸收现象：大数加小数时小数被忽略
  • 抵消现象：相近数相减导致精度丢失
  • 非结合性：(a + b) + c ≠ a + (b + c)

你对C++中的新特性了解吗？比如说C11的新特性？

  1. 自动类型推导（auto）： 让编译器自动推导变量类型，简化代码，特别是在迭代器和模板编程中。
  2. 智能指针（unique_ptr, shared_ptr, weak_ptr）： 用于自动管理动态内存，有效防止内存泄漏。
  3. 基于范围的for循环： 简化了对容器元素的遍历操作。
  4. 右值引用和移动语义： 允许资源（如动态数组）的所有权转移，而非深拷贝，极大地提升了性能，特别是在返回容器对象或交换操作时。
  5. Lambda表达式： 可以定义匿名函数对象，与STL算法（如sort、find_if）配合使用非常方便，使代码更简洁清晰。
  6. nullptr： 替代NULL，提供了明确的空指针常量。

struct 与 union 的区别是什么？

在C++中，结构体(struct)和联合体(union)都是*用户定义的类型，但结构体所有成员同时存在且占用独立内存空间

而联合体所有成员共享同一内存空间，仅能存储一个成员的值。

C++11后，联合体可以包含非POD类型成员，并支持成员函数和特殊成员函数。

特性 结构体 (struct) 联合体 (union)
访问权限 默认public 总是public
成员类型 可包含任意类型的成员函数 C++11起可包含成员函数，不能包含引用和虚函数
面向对象特性 支持继承和多态 不支持继承
内存分配 各成员独立内存空间 所有成员共享内存空间
总大小 各成员大小之和+对齐填充 最大成员的大小+对其填充
生命周期 自动管理成员生命周期 非POD成员需要手动管理
数据安全 高，成员互不影响 低，修改一个成员会影响其他

struct 和 class 有哪些区别？

  • 在C++中，struct和class的唯一区别是默认的访问控制权限：
    • struct默认的成员访问权限是public，默认的继承方式也是public。
    • class默认的成员访问权限是private，默认的继承方式也是private。
  • 除此之外，两者在功能上完全等价。但是，通常在代码中我们会有一些约定俗成的用法：
    • struct常用于表示简单的数据结构，例如只包含一些数据成员，或者只有少量的成员函数。
    • class常用于表示具有复杂行为和封装性的类。

静态局部变量，全局变量，局部变量的特点，以及使用场景？

  • 局部变量：
    • 定义在函数内部或代码块内部，生命周期从声明开始到所在作用域结束，存储在栈上
    • 使用场景：用于函数内部的临时计算，不需要在函数调用之间保持值的场景。
  • 全局变量：
    • 定义在函数外部，整个程序可见，生命周期为整个程序，存储在静态存储区。
    • 使用场景：当多个函数需要共享同一个数据，或者数据需要在整个程序运行期间存在时使用。但应谨慎使用，因为全局变量会破坏封装性，增加耦合度。
  • 静态局部变量：
    • 在函数内部用static修饰的局部变量，生命周期为整个程序，但作用域仍为函数内部。存储在数据段。
    • 使用场景：当需要在函数调用之间保持某个变量的值，但又不想使用全局变量时使用。例如，用于统计函数被调用的次数。

指针和引用的区别？

  • 指针是一个变量，存储的是另一个变量的地址；引用是一个别名，是另一个变量的同义词。
  • 指针可以被重新赋值指向不同的变量；引用一旦初始化后就不能再改变（始终指向初始化时引用的变量）。
  • 指针可以为空（nullptr）；引用必须绑定到一个已有的对象，不能为空（但可以有悬空引用，这是不安全的）。
  • 指针需要解引用（*）来访问所指向的对象；引用直接使用，无需解引用。
  • 指针有自己独立的内存地址；引用和它引用的对象共享同一个地址（即引用本身的地址就是原变量的地址，但实际上引用在底层通常通过指针实现，但这是编译器细节）。
  • 使用场景：指针常用于需要重新指向、可选参数（可为空）的情况；引用常用于函数参数传递（避免复制）和返回值，特别是用于实现运算符重载和函数链式调用。

函数指针如何定义？使用场景有哪些？

  • 定义：函数指针是指向函数的指针。定义时需要指定函数的返回类型和参数类型。
    例如：int (*funcPtr)(int, int); 表示funcPtr是一个指向返回类型为int，参数为两个int的函数的指针。
  • 使用场景：
    • 回调函数：将一个函数作为参数传递给另一个函数，在特定事件发生时调用。例如，在事件驱动编程、排序算法中传递比较函数。
    • 动态函数调用：根据运行时条件选择调用不同的函数。
    • 函数表（跳转表）：用一个函数指针数组来根据索引调用不同的函数，常用于状态机或解释器。

关键字

static 在不同场景下的作用？

  • 在全局变量或函数前：限制作用域为当前文件，避免命名冲突（文件作用域）。
  • 在局部变量前：延长生命周期至整个程序运行期，但作用域不变（函数内）。
  • 在类的成员变量前：该成员为所有类对象共享，属于类而非对象，需要在类外单独定义。
  • 在类的成员函数前：该函数属于类，不能访问非静态成员，只能访问静态成员。

const 关键字在不同场景下的作用？

  • 修饰变量：表示常量，不可修改。
  • 修饰指针：
    • const在*左侧：指向的数据是常量，不可通过指针修改。
    • const在*右侧：指针本身是常量，不可修改指针的指向。
  • 修饰引用：常引用，不能通过引用修改原对象，常用于函数参数，避免拷贝且防止修改。
  • 修饰类成员函数：表示该函数不会修改类的成员变量（mutable修饰的除外）。

constexpr 和 const 的区别是什么？

  • constexpr 是 C++11 引入的，用于声明常量表达式，要求编译时就能计算出值，可以用于变量、函数和构造函数。
  • const 只表示运行时常量，不保证编译时就能确定值（但有时也可以，如用字面量初始化）。
  • constexpr 函数在编译时执行，若参数是常量表达式，则返回常量表达式；否则，可作为普通函数运行。
  • constexpr 变量必须由常量表达式初始化，而 const 变量可以由运行时表达式初始化。

const 和 define 的区别是什么？

  • const 是C++关键字，定义有类型的常量，编译器会进行类型检查，有作用域。
  • define 是预处理指令，进行简单的文本替换，无类型检查，无作用域，容易出错。
  • const 常量在编译时处理，define 在预处理阶段替换。
  • const 定义的常量，在C语言中是存储在内存中、需要额外的内存空间的；define定义的常量，运行时是直接的操作数，并不会存放在内存中。

define 和 typedef 的区别是什么？

  • define 是预处理指令，用于定义宏，可以进行文本替换，包括类型、常量、代码片段等，无作用域，不进行类型检查。typedef 是C/C++关键字，用于为已有类型定义别名，有作用域，编译器会进行类型检查。
  • define 是在编译的预处理阶段起作用，typedef 是在编译、运行的时候起作用
  • define 不分配内存，给出的是立即数，有多少次使用就进行多少次替换；typedef 在静态存储区中分配空间，在程序运行过程中内存中只有一个拷贝

using 和typedef 有哪些区别？

using 是 C++11 特性，若需兼容 C++03/98，需用 typedef；C++11 及以上的 typedef 使用场景都可以用using 替代，且 using 的语法更直观、功能更强大（是 typedef 的超集）

define 和 inline 的区别是什么？

  • define 定义预编译时处理的宏，只是简单的字符串替换，无类型检查，不安全。inline 先将内联函数编译完成生成了函数体直接插入被调用的地方，减少了压栈，跳转和返回的操作。没有普通函数调用时的额外开销
  • 内联函数是一种特殊的函数，会进行类型检查；
  • inline只是给编译器的建议，最终是否内联由编译器决定。编译器可能因为函数体太大、包含循环、递归调用等原因拒绝内联。相反，没有inline标记的函数也可能被编译器自动内联。
  • C++中inline编译限制：
    • 不能存在任何形式的循环语句
    • 不能存在过多的条件判断语句
    • 函数体不能过于庞大
    • 内联函数声明必须在调用语句之前

final 关键字的作用？

  • 修饰类：final修饰的类不可被继承，是最终类
  • 修饰方法：final修饰的方法不能被重写
  • 修饰变量：
    • 修饰基本数据类型的变量，其数值在初始化之后便不能修改，为常量。
    • 修饰引用类型的变量，在对其初始化之后便不能再让其指向另一个对象。虽然不能再指向其他对象，但是它指向的对象的内容是可变的。

final 在性能上有什么好处？

有。编译器如果知道一个类是 final 的，或者一个虚函数是 final 的，它就可以进行去虚拟化 (Devirtualization) 优化。

这意味着编译器可以在编译时确定调用的是哪个具体函数，从而将虚函数调用（通过虚表指针间接调用）优化为普通的直接函数调用，甚至内联调用。

这消除了间接跳转的开销，对性能关键代码非常有益。

extern 关键字有什么用？

  • 声明外部变量/函数：表示定义在其他文件中，当前文件只是引用
  • C++调用C代码：extern “C” 按C语言规则编译函数（避免名称修饰）
  • 模板显式实例化声明：extern template 告诉编译器实例化在其他文件

extern “C”的作用是什么？

extern “C” 是 C++ 编译器提供的一个链接规范（Linkage Specification），其主要作用是指示编译器按照 C 语言的规则来编译和链接其修饰的代码。

这主要体现在函数名修饰（Name Mangling）和调用约定（Calling Convention）上，目的是为了解决 C++ 与 C 以及其他语言在混合编程时的链接兼容性问题。

简单点说： extern “C” 用于告诉 C++ 编译器，被它修饰的函数或变量应使用 C 语言的编译和链接规则，即不进行名称修饰（Name Mangling），从而确保 C++ 代码可以正确链接到用 C 语言编译的库或目标代码。

volatile 的作用是什么？

volatile的主要作用是：禁止编译器对该变量进行优化（如缓存到寄存器、消除“冗余”读写操作），确保每次访问变量时都直接从其内存地址中读取或写入。

sizeof 和 strlen 的区别是什么？

  • sizeof 是 C++ 中的一个编译时一元运算符，用于计算一个类型或对象在内存中所占的字节数，其结果在编译期就已确定。
  • strlen 是 C 标准库中的一个运行时函数，它接收一个以空字符 \0 结尾的字符串的起始地址，遍历内存直到遇到 \0，返回其间的字符个数（不包括 \0）。

核心区别：sizeof关心的是内存布局和占用，它是一个静态的、编译期的概念。而strlen关心的是字符串的内容和逻辑长度，它是一个动态的、运行时的概念。

一个常见的误区是认为sizeof(数组)可以得到数组的元素个数。这只对在作用域内定义的原始数组有效，对于已退化为指针的数组形参，sizeof得到的是指针的大小。

sizeof 如何获取变量或类型的大小？

编译期运算符：返回类型或对象的内存字节数
使用方式：

  • sizeof(type)：获取类型大小
  • sizeof expr：获取表达式结果类型大小（不计算表达式）
    • 特点：在编译期确定，可用于数组大小计算等

new 和 malloc 有什么区别？

  • new 是C++运算符，malloc 是C库函数
  • new 在堆上分配内存并调用构造函数，malloc 只在堆上分配内存
  • new 返回类型指针，malloc 返回 void*
  • 失败时new 抛出 std::bad_alloc，malloc 返回 NULL
  • new/delete 可重载，malloc/free 不可
  • new 自动计算大小，malloc 需手动指定大小

delete 和 free 有什么区别？

  • delete是C++运算符，free 是C库函数
  • delete 调用析构函数并释放内存，free 只释放内存
  • delete[] 释放数组，free 不区分
  • 两者对空指针都安全（无操作）

nullptr 和 NULL 有什么区别？

  • nullptr 是 std::nullptr_t 类型，NULL 通常是整数0或 (void*)0
  • nullptr 可精确匹配指针类型，NULL 可能匹配整型
  • nullptr 避免误用为整数，更安全

介绍一下四种强制类型转换？

  • static_cast：相关类型转换（基本类型、父子类指针、void*）
    • 进行上行转换（子转父）是安全的，进行下行转换（父转子）是不安全的
  • dynamic_cast：带类型检查的 多态父 转子（需多态，失败返回nullptr）
    • 一般用于安全向下转换（多态基类转化为派生类）
    • 仅适用于包含虚函数的类（多态类）
  • const_cast：去掉或加上const/volatile属性
  • reinterpret_cast：底层二进制重解释操作
    • 仅在编译期对二进制位做 “暴力重解释”
    • 不做任何类型检查、内存布局调整或数据转换，完全依赖程序员保证类型的二进制兼容性
    • 我没用过，也没见人用过，也不建议用

i++ 与 ++i 有什么区别？

  • i++ 返回旧值，++i 返回新值
  • ++i 通常更高效（不需保存临时值）
  • 重载时，前缀++返回引用，后缀++返回副本
