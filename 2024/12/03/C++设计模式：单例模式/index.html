C++设计模式：单例模式

发表于2024-12-03|更新于2025-12-22|C++C++设计模式
|总字数:1.4k|阅读时长:4分钟|浏览量:

VERSION：1.0.0

单例模式是 C++ 中最常用的设计模式之一，它确保一个类仅有一个实例，并提供一个全局访问点来获取该实例。这种模式适合用于资源管理器、日志系统、配置中心等需要全局唯一实例的场景。

单例模式的核心要点

  1. 私有构造函数：防止外部通过new创建实例
  2. 私有静态成员变量：存储类的唯一实例
  3. 公有静态成员函数：提供全局访问点，返回唯一实例
  4. 禁用拷贝和赋值：防止通过拷贝构造或赋值运算符创建新实例

常见实现方式

单例模式主要有两种经典实现：饿汉式和懒汉式，两者的核心区别在于实例创建的时机。

饿汉式（Eager Initialization）

特点：程序启动时（类加载阶段）就创建实例，线程安全，但可能提前占用资源。

懒汉式（Lazy Initialization）

特点：第一次使用时才创建实例，节省资源，但需要处理线程安全问题（C++11 及以上标准中，局部静态变量的初始化是线程安全的，可以利用这一特性实现简洁的线程安全懒汉式单例）。

单例模式的优缺点

  • 优点：
    • 控制实例数量，节省系统资源
    • 提供全局访问点，方便统一管理
  • 缺点：
    • 可能导致代码耦合度升高
    • 不利于单元测试（难以模拟单例对象）
    • 破坏单一职责原则（既负责业务逻辑，又负责实例管理）

注意事项

  1. 必须禁用拷贝构造和赋值运算符，否则可能通过Logger copy = Logger::getInstance();创建新实例。
  2. 懒汉式在 C++11 前需手动加锁（如std::mutex）保证线程安全，C++11 后可直接使用局部静态变量。
  3. 单例的生命周期与程序一致，适合管理长期存在的资源（如日志、配置）。
  4. 构造函数的执行时机
  • 饿汉式：
    构造函数在程序启动阶段（main 函数运行前） 就会被调用，因为类的静态成员初始化早于 main 函数。
    即使程序全程没有调用getInstance()，实例也已经创建。
  • 懒汉式：
    构造函数在第一次调用getInstance()时才会被调用，完全按需创建。
    如果程序从未使用该单例，构造函数不会执行，节省资源。
  1. 线程安全的实现差异
  • 饿汉式：
    天然线程安全（静态成员初始化在程序启动时由主线程完成，无多线程竞争），无需额外加锁。
  • 懒汉式：
    C++11 之前需要手动加锁（如std::mutex）保证线程安全；
    C++11 及以上利用 “局部静态变量初始化线程安全” 的特性，无需手动加锁，但代码中仍能看到getInstance()内定义静态变量的特征。
