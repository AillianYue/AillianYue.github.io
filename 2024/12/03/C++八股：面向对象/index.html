C++八股：面向对象

你对面向对象了解吗？介绍一下你对它的认识

面向对象编程是一种以对象为中心的编程范式，其核心思想是封装、继承、多态。

  • 封装：将数据（属性）和操作数据的方法（函数）捆绑在一起，并对外隐藏内部实现细节，只提供可控的接口。通过class关键字定义类，使用private、protected、public访问说明符来控制成员的可见性。
  • 继承：允许基于已有类创建新类，新类继承父类的特性和行为，并可以添加自己的新特性,实现了代码的复用和层次化分类。使用: public/protected/private BaseClass语法。
  • 多态：允许不同类的对象对同一消息做出不同的响应,提高了代码的抽象性和可扩展性。通过虚函数机制实现。在基类中将函数声明为virtual，在派生类中进行override。程序运行时，通过基类指针或引用调用虚函数，会根据指针实际指向的对象类型来调用相应的函数。

为什么要封装、多态、继承？

  • 为什么要封装？
    • 隐藏实现细节，降低复杂度：使用者只需关注接口，无需关心内部实现，使得代码更易理解和维护。
    • 提高安全性：通过将数据设置为私有，可以防止外部代码随意修改内部数据，只能通过受控的方法进行访问和修改，保证了数据的一致性和有效性。
    • 降低耦合度：只要公开的接口不变，内部实现的修改不会影响使用该对象的其他代码。
  • 为什么要继承？
    • 代码复用：这是最直接的目的。子类可以复用父类已经定义好的属性和方法，避免了重复编写相同的代码。
    • 建立类之间的层次关系：通过继承，可以清晰地表示现实世界中“是一类”的关系，使代码结构更符合逻辑，更易于扩展。
  • 为什么要多态？
    • 增强程序的可扩展性：这是多态最重要的价值。
    • 接口统一：允许用父类的指针或引用来操作子类对象，使代码更通用、更简洁。

总结： 封装是基础，用于隐藏细节，保证安全；继承是扩展，用于代码复用，建立层次；多态是目标，基于继承实现，用于提高代码的灵活性和可扩展性。三者共同构成了面向对象编程的强大能力。

聊一聊C++中的多态？

在C++中，多态是面向对象编程（OOP）的三大核心特性之一（另外两个是封装和继承），其核心思想是：同一操作作用于不同对象时，可产生不同的行为效果。

C++中的多态分为两类：静态多态（编译时多态）和动态多态（运行时多态）

  • 静态多态是在编译阶段就确定具体行为的多态，主要通过函数重载和运算符重载实现。
  • 动态多态是在程序运行阶段才确定具体行为的多态，其核心是虚函数（Virtual Function），通过基类指针或引用调用派生类的重写函数实现。

静态绑定和动态绑定是什么？

在C++中，绑定(Binding)是将函数调用与函数实现关联起来的过程。根据绑定发生的时机，可以分为静态绑定和动态绑定。

静态绑定 (Static Binding)

静态绑定也称为早期绑定(Early Binding)，发生在编译阶段。编译器在编译时根据调用函数的对象类型或指针类型来确定调用哪个函数。

特点：

  • 在编译时解析函数调用
  • 基于对象或指针的声明类型
  • 效率更高，没有运行时开销
  • 适用于非虚函数、重载函数和模板函数

动态绑定 (Dynamic Binding)

动态绑定也称为晚期绑定(Late Binding)，发生在运行时。通过虚函数(virtual functions)和继承实现，允许程序根据实际对象类型调用相应的函数。

特点：

  • 在运行时解析函数调用
  • 基于对象的实际类型
  • 通过虚函数表(vtable)实现
  • 是实现多态性的关键机制

什么是构造函数和析构函数？

构造函数：与类同名、无返回类型的特殊成员函数，在创建对象时自动调用，用于初始化对象状态。

析构函数：类名前加~的特殊成员函数，在对象销毁时自动调用，用于清理资源。

什么是虚函数和虚函数表？

虚函数：使用virtual关键字声明的成员函数，支持动态绑定（运行时多态）。

虚函数表：每个多态类都有一个虚函数表，是函数指针的静态数组，存储该类所有虚函数的地址。

虚函数表的机制是什么？

虚函数表是 C++ 实现运行时多态（动态绑定）的核心机制。

当一个类中含有virtual关键字声明的虚函数时，编译器会为该类生成一个虚函数表，这是一个函数指针数组，数组中的每个元素指向一个虚函数的实际代码地址。同时，编译器会隐式地在类的每个对象实例中添加一个指针，称为虚函数表指针，它指向该类的虚函数表。

当发生继承时，子类会继承父类的虚函数表，如果子类重写了父类的虚函数，那么子类虚函数表中对应的函数指针会被更新为指向子类自己的函数实现。

详细版：

虚函数的机制（实现）依赖两个关键结构：

  1. 虚函数表（vtable）
  • 本质：每个包含虚函数（或继承自含虚函数的类）的类，都会在编译时被编译器分配一个静态数组，即虚函数表。
  • 内容：存储该类所有虚函数的函数地址（函数指针）。
  • 特点：
    • 每个类只有一个虚函数表（全局唯一，属于类级别的资源）。
    • 派生类会继承基类的虚函数表：
      • 若派生类重写了基类的虚函数，派生类虚函数表中对应位置会被替换为派生类自己的函数地址。
      • 若派生类新增虚函数，这些函数地址会被追加到派生类虚函数表的末尾。
  1. 虚函数指针（vptr）
  • 本质：每个包含虚函数的类的对象，都会隐含一个指向该类虚函数表的指针成员（通常是对象的第一个成员）。
  • 初始化：对象在构造时，编译器会自动将vptr初始化，使其指向所属类的虚函数表。
  • 特点：
    • 每个对象都有一个vptr，因此会增加对象的内存开销（一个指针的大小：32位系统4字节，64位系统8字节）。
    • vptr是编译器自动添加的，用户无法直接访问（但可通过指针操作间接获取）。

虚函数调用的完整流程是什么？举例说明

当通过基类指针/引用调用虚函数时，底层执行步骤如下：

  1. 通过对象的vptr找到其所属类的vtable。
  2. 在vtable中根据函数的索引（编译时确定）找到目标虚函数的地址。
  3. 调用该地址对应的函数。

以上过程在运行时完成，因此实现了“根据对象实际类型调用函数”的动态绑定。

举例：以Base和Derived为例，其虚函数表结构如下：

  • Base类：
    • 虚函数表（Base::vtable）中存储 &Base::func。
    • Base对象的vptr指向 Base::vtable。
  • Derived类：
    • 继承Base的虚函数表，但由于Derived重写了func，因此Derived::vtable中存储 &Derived::func（替换了基类的函数地址）。
    • Derived对象的vptr指向 Derived::vtable。

当执行ptr->func()时：

  • ptr指向Derived对象，因此通过ptr访问到的vptr指向Derived::vtable。
  • 从Derived::vtable中找到func的地址（&Derived::func），最终调用Derived::func。

为什么构造函数不能是虚函数？

虚函数调用依赖于对象的虚表指针（vptr），而虚表指针是在构造函数中初始化的。在构造函数执行期间，对象类型还不完整，无法确定调用哪个虚函数实现。

为什么要使用虚析构函数？

为了确保通过基类指针删除派生类对象时，能够正确调用派生类的析构函数，从而避免资源泄漏。

哪些函数不能被声明为虚函数？

  1. 构造函数。
  2. 静态成员函数：静态函数属于类本身，而不是类的对象，因此没有this指针，也无法访问虚表。
  3. 内联函数（通常）：内联函数在编译时展开，而虚函数的行为是在运行时确定的。但是，如果虚函数在类内定义，编译器可能会忽略inline请求，将其视为普通的虚函数。
  4. 友元函数：友元函数不是类的成员函数，因此不能声明为虚函数。
  5. 普通函数（非成员函数）：虚函数必须是类的成员函数。

使用虚函数时应该注意什么？

  1. 一个类被设计为基类（即可能被继承）时，应该将析构函数声明为虚函数。这是为了确保当通过基类指针删除派生类对象时，能够正确调用派生类的析构函数，从而避免资源泄漏。
  2. 构造函数不能是虚函数：对象构造时vptr尚未初始化，无法动态绑定。
  3. 静态成员函数不能是虚函数：静态函数属于类，不依赖对象，没有this指针，无法通过vptr调用。
  4. 内联函数不能是虚函数：内联函数在编译时展开，而虚函数需要运行时绑定，两者冲突（编译器会忽略inline关键字）。
  5. 含有纯虚函数的类（抽象类）不能实例化对象（无法创建抽象类的实例），只能作为基类被继承，派生类必须重写所有纯虚函数才能实例化（否则派生类仍是抽象类）。
  6. 派生类重写虚函数时，必须保证函数名、参数列表、返回值完全一致。

什么是抽象类和纯虚函数？

  • 纯虚函数：在基类中声明但没有定义的虚函数，其声明方式为在函数声明后加上= 0。
  • 抽象类：包含至少一个纯虚函数的类。抽象类不能实例化，只能作为基类使用。派生类必须实现所有纯虚函数，否则派生类也会成为抽象类。

虚函数和纯虚函数有什么区别？

特性 虚函数 纯虚函数
语法 virtual void func(); virtual void func() = 0;
实现 基类必须提供实现 基类可以不提供实现（C++11后可提供）
抽象类 不强制类为抽象类 强制类为抽象类（不能实例化）
派生类要求 可重写，也可不重写 必须重写（除非派生类也是抽象类）
接口定义 提供默认实现 定义接口规范
设计意图 可选的多态行为 强制接口实现

C++ 中 this 指针的原理及存储位置是什么？

在C++中，this指针是一个隐含的指针，它指向调用该成员函数的那个对象的地址。它用于在成员函数内部访问对象自身的成员。

  • 存储位置：this 指针本身是一个函数形参，因此，它和其他函数参数一样，存储在栈区（对于x86/ARM等常见架构）。
  • 生命周期：this 指针的生命周期与任何其他函数参数的生命周期相同，即在函数被调用时创建，在函数返回时销毁。
  • 类型：在非 const 成员函数中是 ClassName* const，在 const 成员函数中是 const ClassName* const
  • 用途：主要用于解决成员变量与函数参数同名时的歧义，以及在成员函数中返回对象自身的引用（return *this;），这在链式调用中非常常见。

重载、重写和重定义的区别是什么？

  • 重载：同一作用域，函数名相同，参数不同，编译时多态。
  • 重写：派生类重写基类中的虚函数，具有相同的函数名、参数列表和返回类型，运行时多态。使用virtual 和 override 关键字。
  • 重定义：派生类重定义基类中的非虚函数。这会隐藏基类同名函数。可通过作用域解析运算符来访问基类的同名函数。
