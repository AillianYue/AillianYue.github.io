C++八股：STL

|总字数:4.8k|阅读时长:15分钟|浏览量:

VERSION：1.0.0

你对STL有哪些了解？

  • STL是C++标准模板库，采用模板编程技术（泛型编程），通过迭代器将数据结构（容器）与算法（算法）连接起来，实现了代码与数据类型的分离。
  • STL包含了六大组件：
    • 容器：
      • 顺序容器（Sequence Containers）：vector, list, deque, array, forward_list
      • 关联容器（Associative Containers）：set, map, multiset, multimap
      • 无序容器（Unordered Containers）：unordered_map, unordered_set, unordered_multimap, unordered_multiset
    • 容器适配器：stack、queue
    • 算法： 定义了用于操作容器中元素的函数模板，如排序sort、查找find，这些算法通过迭代器与容器交互，与具体的数据结构解耦。
    • 迭代器： 容器与算法之间的桥梁，提供访问容器元素的统一方法
    • 仿函数： 行为类似函数的对象，可以让算法的行为参数化，变得更加灵活。（如greater<int>）
    • 空间配置器：负责底层内存的分配与管理（通常无需关注）。

vector和array的使用场景分别是什么？

  • array：固定大小的静态数组，在栈上分配内存。
    • 场景：已知元素数量且不变，对性能有极致要求（无动态分配开销），或需要与C风格API交互。
  • vector：动态大小的数组，在堆上管理内存。
    • 场景：元素数量未知或会变化；需要频繁随机访问；存储顺序与添加顺序一致；尾部插入删除频繁。

vector如何保证元素连续存储？

vector底层使用一个动态分配的连续数组来存储元素。其iterator本质上就是原生指针（T*），通过指针的算术运算（如ptr + n）即可在常数时间内访问任意元素，这天然保证了连续性。当vector的内存容量不够时，会重新分配一块更大的内存，再把旧的数组中的数据复制到新的内存块中。

vector底层原理和扩容过程是什么？

  • 原理：维护三个核心指针：start（指向内存首）、finish（指向最后一个元素的下一个）、end_of_storage（指向分配内存的末尾）。
  • 扩容过程：
    1. 当size() == capacity()时，插入新元素会触发扩容。
    2. 分配一块新的、更大的内存（通常是原容量的1.5或2倍，取决于编译器）。
    3. 将旧内存的所有元素移动或拷贝到新内存（C++11后优先使用移动语义）。
    4. 释放旧内存。
    5. 更新三个指针到新内存区。

vector的push_back和emplace_back有什么区别？

  • push_back(const T& value)：接受一个已构造的对象，将其拷贝或移动到容器尾部。
  • emplace_back(Args&&... args)：接受构造T所需的参数包，直接在容器尾部内存处构造对象，无需创建临时对象。
  • emplace_back避免了不必要的临时对象构造和拷贝/移动操作，对于非平凡类型性能更优。两者在功能上等价，但emplace_back更高效。

vector和list有什么区别？

核心区别：底层数据结构的连续性，这直接导致了它们在随机访问和中间插入/删除性能上的根本差异。

  1. vector底层实现是数组；list是双向链表
  2. vector是顺序内存,支持随机访问，list不行
  3. vector在中间节点进行插入删除会导致内存拷贝，list不会
  4. vector一次性分配好内存，不够时才进行翻倍扩容；list每次插入新节点都会进行内存申请
  5. vector随机访问性能好，插入删除性能差；list随机访问性能差，插入删除性能好
特性 vector（动态数组） list（双向链表）
底层数据结构 一段连续的内存空间 非连续的节点，每个节点包含数据和指向前后节点的指针
随机访问 支持，效率高 O(1) 不支持，效率低 O(n)
插入/删除 在尾部快 O(1)，在中间或头部慢 O(n) 在已知位置插入/删除快 O(1)
内存使用 内存紧凑，额外开销小 每个节点都需要额外空间存储指针
迭代器类型 随机访问迭代器 双向迭代器
缓存友好性 好，数据局部性原理 差，内存不连续

为什么vector内存连续，list不连续？

  • vector：设计目标是提供快速的随机访问。连续内存布局允许通过基地址+偏移量直接计算元素地址（指针算术），这是实现O(1)随机访问的物理基础。
  • list：设计目标是实现任意位置的高效插入/删除。采用链式结构（每个节点独立分配，通过指针链接）可以在常数时间内重构链接，无需移动其他元素。连续内存反而会使其优势丧失。

如何判断链表中是否有环？

最经典的方法是使用 快慢指针，也称为 Floyd 判圈算法：

  1. 初始化两个指针slow和fast，均指向头节点。
  2. slow每次走1步，fast每次走2步。
  3. 如果链表中无环，fast会先到达末尾（nullptr）。
  4. 如果链表中有环，fast会先进入环内绕圈，最终slow也会进入环。由于fast相对slow每次多走1步，它们必然会在环内的某一点相遇。
  5. 当slow == fast时，即可判定有环。

list是如何实现元素的插入和删除的？

std::list通常是一个双向链表。每个节点包含：数据、指向前驱节点的prev指针、指向后继节点的next指针。

  • 插入（在迭代器pos位置之前插入）：
    1. 创建一个新节点。
    2. 调整四个指针：
      • 新节点的prev指向pos节点的原前驱。
      • 新节点的next指向pos节点。
      • pos原前驱节点的next指向新节点。
      • pos节点的prev指向新节点。
  • 删除（删除迭代器pos指向的节点）：
    1. 令pos前驱节点的next指向pos的后继节点。
    2. 令pos后继节点的prev指向pos的前驱节点。
    3. 销毁pos节点并释放内存。
  • 核心：这些操作都只涉及指针的常数次修改，时间复杂度为O(1)，不会影响其他元素的位置或迭代器的有效性（除了被删除的那个）。

什么是哈希表？

哈希表是一种使用哈希函数来实现高效数据访问的数据结构。

  • 核心思想： 通过一个哈希函数，将键映射到数组中的一个位置，从而可以在平均O(1)时间复杂度内完成插入、删除和查找操作。
  • 工作流程：
    • 插入： 计算键的哈希值，根据哈希值确定在数组（称为哈希桶）中的存储位置，然后将值存入该位置。
    • 查找： 计算键的哈希值，找到对应的位置，检查该位置的值是否是要查找的键值对。
    • 关键问题： 哈希冲突，即不同的键可能被映射到同一个数组位置上。

哈希映射产生冲突了该怎么办？

  • 方案1：开放定址法：当冲突发生时，通过某种探测方法（如线性探测、平方探测）在哈希表中寻找“下一个”空位置。
    • 优点：所有数据都存储在同一个数组中，结构简单。
    • 缺点：容易产生“聚集”现象，删除操作麻烦。
  • 方案2：链地址法：这是最常用的方法。哈希表的每个位置（桶）不再直接存储一个元素，而是存储一个链表（或红黑树）的头指针。当冲突发生时，将新的键值对插入到对应桶的链表中。
    • 优点： 处理简单，无聚集现象，易于删除。
    • 缺点： 需要额外的指针空间。如果某个链表过长，会退化为 O(n) 的查找。

如何在map和set中查找元素？

  • find 方法：直接查找特定键的值，返回指向该元素的迭代器，如果未找到则返回end()迭代器。
  • count 方法：返回具find 方法，由于键的唯一性，返回值要么是0（未找到），要么是1（找到）。
  • lower_bound 方法：返回指向不小于给定键的首个元素的迭代器。
  • upper_bound 方法：返回指向大于给定键的首个元素的迭代器。
  • equal_range 方法：返回一个迭代器对，包括lower_bound和upper_bound，对于不存在的键，这两个迭代器是相同的。对于唯一键，不会返回相同值

set和map有什么区别？

  • map 存储的是键值对（pair<const K, V>），元素类型为 pair，插入时需要提供键和值（如 insert({key, value})），查找返回的迭代器指向键值对。由于需要存储值，内存占用稍大，典型使用场景是字典或映射关系。
  • set 仅存储键（Key），元素类型即为键本身，插入时只需提供值（如 insert(value)），查找返回的迭代器直接指向键。内存占用较小，常用于去重集合或存在性检查。
  • 两者底层实现相同（通常为红黑树），但 set 可视为只有键的 map，因此操作和性能特征一致，主要差异在于是否关联值。

map、set、multimap、multiset有什么区别？

1.std::map

  • 存储键值对，键是唯一的
  • 排序：map中的元素根据键自动排序。
  • 访问：通过键访问值，使用[]可以直接访问或添加元素。

2. std::set

  • 仅存储键，键是唯一的
  • 排序：set中的元素根据元素值自动排序。
  • 访问：只能通过迭代器访问元素，不支持[]。

3.std::multimap

  • 存储键值对，键可重复
  • 排序：multimap中的元素根据键自动排序，对于具有相同键的元素，它们在内部是有序的（通常是插入顺序）。
  • 访问：通过键访问值，但不能直接访问特定键的所有值，需要遍历。

4.std::multiset

  • 仅存储键，键可重复
  • 排序：multiset中的元素根据元素值自动排序，对于具有相同值的元素，它们在内部是有序的（通常是插入顺序）。
  • 访问：只能通过迭代器访问元素，不支持下标操作符。

容器 map 、deque、list的实现原理？

  • map：红黑树（自平衡二叉搜索树），保证插入、删除、查找均为O(log n)，中序遍历有序。
  • deque：分段的连续数组（双端队列）：
    • 由多个固定大小的数组块（buffer）组成
    • 通过一个映射器（中控数组）管理这些块
    • 支持头尾O(1)插入删除，支持随机访问但比vector慢
  • list：双向链表，每个节点含前后指针，插入删除O(1)，不支持随机访问。

map && unordered_map的区别有哪些？

map 基于红黑树实现，元素按键排序（默认升序），保证查找和插入时间稳定为 O(log n)，支持双向迭代且迭代器稳定。它要求键类型支持 < 比较运算符，内存占用相对较低。

unordered_map 基于哈希表实现，元素无序排列，平均情况下查找和插入时间可达 O(1)，但最坏情况为 O(n)。它仅支持前向迭代，且迭代器在 rehash 时可能失效。要求键类型提供哈希函数和 == 比较运算符，内存占用较高（需维护桶数组）。

C++ STL 中 allocator 的作用是什么？

allocator是STL中负责内存管理的组件，它封装了内存分配和释放的细节，实现了容器与内存管理的解耦。通过模板参数允许自定义内存分配策略，满足特殊的内存需求（如内存池、共享内存等）。

allocator 的主要作用包括：

  1. 内存分配：提供 allocate 方法，用于分配未初始化的内存块，该内存块可以容纳指定数量的对象。
  2. 内存释放：提供 deallocate 方法，用于释放之前分配的内存块。
  3. 对象构造：提供 construct 方法，用于在已分配的内存中构造对象（使用 placement new）。
  4. 对象析构：提供 destroy 方法，用于析构对象，但不释放内存。

迭代器有什么用？

迭代器是STL的核心抽象，作用包括：

  • 统一访问接口：为不同容器提供一致的遍历方式，无需了解容器的内部结构，是容器无关的
  • 算法与容器解耦：算法通过迭代器操作数据，不依赖具体容器
  • 泛型编程基础：支持模板元编程
  • 安全性：比裸指针更安全（部分检查）

C++ STL 中迭代器失效的场景有哪些？

简化版：

  • 迭代器失效是指容器被修改后，旧的迭代器/指针/引用不再有效。
  • 连续存储（vector/string）在 扩容或中间插入/删除 会使大量迭代器失效； 节点式容器（list/map/set）插入一般不失效，erase 只使被删元素失效； 哈希容器在 rehash 时会使 全部 迭代器失效。

详细版：

  • std::vector / std::basic_string（连续存储）
    • push_back/insert/emplace/resize 可能触发扩容 → 所有 迭代器/指针/引用失效
    • 即便不扩容：在插入/删除位置 及其之后 的迭代器失效
    • erase(pos)：从 pos 起至末尾的迭代器失效；返回指向 被删元素后一个 的新迭代器
  • std::deque
    • 在 首/尾插入删除 可能使 全部 迭代器失效（实现相关）
    • 中间插入/删除通常使 受影响的区段 迭代器失效
  • std::list / std::forward_list（链式存储）
    • 插入/拼接 不会使现有元素的迭代器失效
    • 删除 仅使 被删元素 的迭代器失效
  • std::map / std::set（有序关联容器，节点式）
    • insert 不 使已有迭代器失效
    • erase 仅使 被删节点 的迭代器失效
  • std::unordered_map / std::unordered_set（哈希容器）
    • rehash / reserve 导致 所有 迭代器失效
    • insert 一般不使已有迭代器失效（除非触发 rehash）
    • erase 仅使被删元素的迭代器失效
容器类型 插入操作 删除操作
vector/string 可能全部失效（重新分配） 删除点及之后迭代器失效
deque 可能全部失效 删除点及之后迭代器可能失效
list/forward_list 不失效 只有被删除元素的迭代器失效
map/set 不会效 只有被删除元素的迭代器失效
unordered_map/set 可能全部失效（重新哈希rehash） 只有被删除元素的迭代器失效

如何避免迭代器失效？

简化版：

  • 用 erase 返回值继续遍历、reserve 预容量、修改后重新获取迭代器、必要时选用 迭代器稳定的容器。

详细版：

  1. 使用返回值继续遍历
    • erase（C++11 起）返回下一个有效迭代器：it = c.erase(it);
  2. 提前容量规划（连续存储容器）
    • vector/string 在大量插入前 reserve(n)，减少扩容次数
  3. 遍历时尽量不用下标 + 外部缓存迭代器
    • 修改容器后 重新获取 相关迭代器/指针/引用
  4. 选择“节点稳定”的容器
    • 频繁插删且需要稳定迭代器：优先 list、(unordered_)map/set
  5. 注意哈希容器的 rehash
    • 批量插入前 reserve；rehash 后 全部迭代器作废
  6. 使用“索引遍历”替代“迭代器遍历”（当容器允许且你可控制索引失效）
    • 对 vector 的尾删可用索引；警惕插入/删除导致的元素移动
  7. 不要持有临时容器元素的引用/迭代器
    • 返回局部容器的迭代器/引用 → 悬垂
  8. 范围 for + 拷贝值
    • 如只读数据且容器会变更结构，考虑先拷贝必要的值到临时数组再处理

unordered_map 的负载因子与 rehash 机制是什么？

负载因子 = size() / bucket_count()。size()是容器中键值对的数量，bucket_count()是底层桶数组的大小。

负载因子是衡量哈希表拥挤程度的指标。

当负载因子超过预设的最大负载因子时，容器会自动执行 rehash：创建一个新的、更大的桶数组，并将所有现有元素重新哈希到新的数组中，以降低每个桶的平均元素数，从而保证 O(1) 平均时间复杂度的操作性能。

最大负载因子：

  • unordered_map有一个默认的最大负载因子（通常为 1.0）。当前负载因子 > 最大负载因子时，容器会自动触发 rehash 操作，以维持操作的效率。

rehash机制：

  • 当哈希表的负载因子超过最大负载因子时，容器会自动增加桶的数量（通常至少翻倍）并重新将元素分布到新的桶中，这个过程称为rehash。rehash会重新计算每个元素的哈希值，然后根据新的桶数量将其放入对应的桶中。

rehash的触发条件：

  1. 被动触发：当插入操作导致负载因子超过最大负载因子时。
  2. 主动触发：调用 rehash 或 reserve 成员函数时。

过程：

  • 分配一块新的、更大的内存空间作为新的桶数组。
  • 遍历旧桶数组中的所有元素（包括每个桶链表/树上的所有元素）。
  • 根据元素键的新哈希值和新桶数组的大小，重新计算每个元素应该归属的新桶的位置。
  • 将元素移动到新的桶中，释放旧桶数组的内存。

特点：

  • rehash 是一个相对耗时的操作，时间复杂度为 O(n)，其中 n 是容器中元素的数量。
  • 会使所有迭代器失效（但指向元素本身的引用和指针不会失效）。

unordered_map 的 reserve和 rehash有什么区别？

reserve(n)是从元素数量的角度出发，保证在存入 n 个元素前不会 rehash。它内部会计算需要的桶数。rehash(n)是直接从桶数量的角度出发，设置桶的数量至少为 n。通常更推荐使用 reserve，因为它的语义（预留 n 个元素的空间）更直观。

你对C++中的排序算法了解吗？比如说快排，介绍一下你对它的认识。

  • c++标准库中的std::sort算法通常采用IntroSort（内省排序），它是快速排序、堆排序和插入排序的混合算法，平均和时间复杂度为O(N log N)，且能避免快排的最坏情况O(n²)。
  • 快排是一种经典的分而治之策略算法。
    • a.选择基准： 从序列中挑一个元素作为“基准”。
    • b.分区操作： 重新排列序列，所有比基准小的放在前面，比基准大的放在后面，分区完成后，基准就位于序列的中间位置。
    • c.递归排序： 递归地将小于基准的子序列和大于基准的子序列进行快速排序。
  • 它的优点是平均性能非常好，是原地排序（只需要很小的辅助栈空间），缓存友好。
  • 缺点是稳定性差（相等元素的相对位置可能改变），且在最坏情况下（如序列已有序）性能较差。
