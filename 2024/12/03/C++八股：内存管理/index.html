聊一聊C++的智能指针？

为什么需要智能指针？

在原生指针（Raw Pointer）的世界里，程序员需要手动管理内存：

  1. 使用 new 分配内存。
  2. 使用 delete 释放内存。

这很容易出错：

  • 内存泄漏（Memory Leak）：忘记 delete。
  • 悬空指针（Dangling Pointer）：提前 delete 了仍在使用的内存。
  • 双重释放（Double Delete）：对同一块内存 delete 两次。

智能指针的核心思想：将动态分配的内存的生命周期与一个对象（智能指针）绑定。这个对象在析构时，会自动释放其管理的内存。这利用了 C++ 的 RAII (Resource Acquisition Is Initialization) 原则。

智能指针有哪几种？分别介绍一下？

C++ 标准库在 <memory> 头文件中提供了以下几种主要的智能指针：

  1. std::unique_ptr (独占所有权指针)
  2. std::shared_ptr (共享所有权指针)
  3. std::weak_ptr (弱引用指针)
  4. std::auto_ptr (已弃用，C++17 中移除，切勿使用)

std::unique_ptr

特点：

  • 独占：同一时间只有一个 unique_ptr 可以拥有一个给定的对象。
  • 移动语义：所有权可以通过 std::move 转移给另一个 unique_ptr，但无法复制。
  • 轻量高效：开销极小，通常与裸指针相同。
  • 自定义删除器：支持指定自定义的释放逻辑（例如用于 FILE*, DLL 句柄等）。

适用场景：当你想要一个对象在其当前作用域结束时（例如函数返回、块结束）被自动删除，并且不需要多个指针共享所有权时。这是最常用、默认应优先考虑的智能指针。

std::shared_ptr

特点：

  • 共享：多个 shared_ptr 可以共同拥有同一个对象。
  • 引用计数：内部维护一个引用计数器。每拷贝一个 shared_ptr，计数加一；每析构一个 shared_ptr，计数减一。当计数变为 0 时，自动删除管理的对象。
  • 可复制：支持拷贝构造和拷贝赋值。
  • 开销稍大：需要维护控制块（包含引用计数等）。

适用场景：当多个部分 of your code 都需要访问同一个对象，并且你无法确定哪个部分最后使用完这个对象时。

std::weak_ptr

特点：

  • 不拥有所有权：weak_ptr 指向一个由 shared_ptr 管理的对象，但不增加其引用计数。
  • 解决循环引用：主要用于打破 shared_ptr 之间的循环引用问题（例如：父节点持有子节点的 shared_ptr，子节点又持有父节点的 shared_ptr），否则会导致内存泄漏。
  • 检查有效性：必须通过 lock() 方法将其转换为一个临时的 shared_ptr 来访问对象，以此检查对象是否还存在。

make_xxx

创建智能指针优先使用 std::make_unique (C++14) 和 std::make_shared (C++11)：

优点：

  1. 异常安全：避免了因函数参数求值顺序可能导致的内存泄漏。
  2. 性能提升（对 make_shared）：make_shared 通常只需一次内存分配（将对象本身和控制块分配在一起），而 new + shared_ptr 构造函数需要两次。
  3. 代码简洁：不需要重复写类型，更清晰。

聊一聊C++的内存管理？

一个典型的 C++ 程序在内存中（从低地址到高地址）会包含以下部分：

区域 存放内容 特性 管理方式
栈 (Stack) 函数参数、局部变量、函数返回地址等 自动分配和释放，速度快，容量有限（通常几MB）。 编译器自动管理
堆 (Heap) 动态分配的内存（malloc, new 等） 手动分配和释放，容量大（受限于系统虚拟内存），速度慢，使用不当易产生内存泄漏和碎片。 程序员手动管理
全局/静态存储区 全局变量、静态变量（static）、字符串常量等 在程序编译期分配，整个程序生命周期都存在。分为.data（已初始化）和.bss（未初始化）段。 编译器/系统管理
常量区 字符串常量和其他常量（如 const 全局变量） 只读，试图修改会导致段错误（Segmentation Fault）。 编译器/系统管理
代码区 程序的二进制机器指令（函数体的代码） 只读，共享的（多个实例可共享同一份代码）。 编译器/系统管理

栈 (Stack)

  • 功能：用于函数调用。每次调用函数时，会在栈上分配一个“栈帧”(Stack Frame)，用于存放该函数的参数、局部变量以及返回地址。函数执行完毕时，其栈帧被自动销毁。
  • 优点：分配和释放由编译器自动完成，极其高效。
  • 缺点：空间有限，大小在编译时通常就已确定（或设置）。分配过大（如大数组）或过深递归会导致栈溢出 (Stack Overflow)。

堆 (Heap) / 自由存储区 (Free Store)

  • 功能：用于程序运行期间动态申请任意大小的内存。这块内存的生命周期不完全受作用域限制，需要程序员显式控制。
  • C 风格：使用 malloc(), calloc(), realloc() 分配，使用 free() 释放。
  • C++ 风格：使用 new 和 delete (或 new[] 和 delete[]) 操作符。
  • 优点：空间大，分配灵活。
  • 缺点：
    • 内存泄漏 (Memory Leak)：如果分配后忘记释放，这块内存就无法被再次使用。
    • 野指针 (Dangling Pointer)：释放后未将指针置空，再次使用会导致未定义行为。
    • 碎片化 (Fragmentation)：频繁分配和释放不同大小的内存块会导致内存碎片，降低利用率。

技术细节：严格来说，堆 (Heap) 是 C 语言和操作系统术语，由 malloc 管理；自由存储区 (Free Store) 是 C++ 术语，由 new 管理。但在大多数实现中，new 底层是调用 malloc 来实现的，所以它们通常指的是同一块区域。但在语言标准上，这是两个不同的概念。

全局/静态存储区 (Global/Static Storage)

  • 功能：存储全局变量和静态变量。
  • .data 段：存放已初始化的全局变量和静态变量。
  • .bss 段：存放未初始化的全局变量和静态变量（在程序开始执行前会被系统自动初始化为零）。
  • 生命周期：在 main 函数开始前分配，在 main 函数结束后释放。

常量区 (Constant Data Segment)

  • 功能：存放字符串常量和其他 constexpr 或常量数据。
  • 特性：只读。任何试图修改的操作都会引发段错误。

代码区 (Text Segment)

  • 功能：存放程序执行代码（机器指令）的区域。
  • 特性：只读、共享。多个运行同一程序的进程可以在内存中共享一份代码段的副本，节省内存。

堆与栈的区别是什么？

  1. 关于栈（Stack）
  • 在绝大多数现代系统中（如x86, x86-64, ARM），栈是向下生长的，即从高地址向低地址扩展。当一个函数被调用时，它的局部变量等信息被“压栈”（push），栈指针（SP, Stack Pointer）的值会减小。
  • 栈通常被分配在进程虚拟地址空间的高位区域（虽然不是最高位），有一个固定的起始地址（栈底），随着压栈操作，栈顶向低地址方向移动。
  • 栈的大小是有限的。这个大小在线程创建时就被确定了。如果栈空间耗尽（比如过度递归或非常大的局部数组），会导致“栈溢出”（Stack Overflow）错误。
  1. 关于堆（Heap）
  • 堆总体上是从低地址向高地址扩展的。当你使用 new 或 malloc 申请内存时，堆管理器会在现有的堆空间中寻找空闲块来满足请求。如果空间不足，它会通过系统调用（如 sbrk 或 mmap）向操作系统申请扩大堆的大小，这个过程确实是向高地址发展的。
  • 堆的大小受限于系统的虚拟内存大小，通常远大于栈（可达GB甚至TB级别）。它的生命周期由程序员手动控制（new/delete 或 malloc/free），非常灵活，但也容易导致内存泄漏等问题。
  • 堆的“扩展”方向是宏观上的。在其内部，内存的分配和释放是不连续的。堆管理器维护一个“空闲链表”，分配内存时从链表中找到合适大小的块，释放时再将其回收到链表中。因此，堆的内部是碎片化的，程序员先后申请的两块内存的地址高低顺序没有必然关系，不一定是严格由低到高。

new 和 malloc 有什么区别？

  • new 是C++运算符，malloc 是C库函数
  • new 在堆上分配内存并调用构造函数，malloc 只在堆上分配内存
  • new 返回类型指针，malloc 返回 void*
  • 失败时new 抛出 std::bad_alloc，malloc 返回 NULL
  • new/delete 可重载，malloc/free 不可
  • new 自动计算大小，malloc 需手动指定大小

delete 和 free 有什么区别？

  • delete是C++运算符，free 是C库函数
  • delete 调用析构函数并释放内存，free 只释放内存
  • delete[] 释放数组，free 不区分
  • 两者对空指针都安全（无操作）

C++内存管理常用方法有哪些？

  1. 静态内存分配： 在编译期确定大小和生命周期。例如全局变量、static变量，内存在程序启动时分配，结束时释放。
  2. 栈内存分配： 函数内的局部变量、函数参数等。内存的分配和释放由编译器自动管理，遵循后进先出的原则，效率高，但容量有限。
  3. 堆内存分配： 使用new/malloc等操作符在运行时动态申请任意大小的内存，生命周期由程序员手动控制，使用 delete/free释放，非常灵活，但管理不当易产生内存泄漏或野指针。
  4. 智能指针： 这是现代 C++ 推荐的管理堆内存的方，它是类模板，通过 RAII 机制将堆内存封装在对象中，利用栈上对象的析构函数自动释放内存。主要类型有：
    1. std::unique_ptr： 独占所有权，同一时间只能有一个 unique_ptr指向一个对象。
    2. std::shared_ptr： 共享所有权，通过引用计数管理多个指针指向同一个对象。
    3. std::weak_ptr： 配合 shared_ptr使用，解决循环引用问题，不增加引用计数。

内存泄漏如何排查？

  1. 重现问题：确定能够稳定复现泄漏场景
  2. 初步分析：依赖经验（资源未析构、发生异常等问题）或使用Coverity、Valgrind等工具初步定位
  3. 代码审查：检查或注释可疑代码段
  4. 增量测试：逐步锁定泄露位置并修复
  5. 修复验证：修复后再次验证

堆和栈了解吗？说一下它们的底层区别。

  • 栈： 由操作系统自动管理，函数调用时的局部变量、参数都在栈上分配，函数结束自动释放。分配效率极高，但空间有限且大小编译时常确定。
  • 堆： 由程序员手动管理，通过malloc/free或new/delete申请和释放，空间很大，分配灵活，但容易产生内存碎片，分配速度也比栈慢在资源受限的嵌入式系统中，需要谨慎使用堆，避免内存泄漏。

如何避免内存碎片？

内存碎片分为外部碎片（空闲内存分散在不连续的小块中，无法满足大分配请求）和内部碎片（分配的内存块内部未被使用的部分）。

避免策略是一个系统工程，需要从代码习惯、库的使用到自定义分配器等多个层面考虑。

  1. 减少动态内存分配：
    1. 根本法则：最好的避免就是不做，优先使用栈内存、静态存储期对象或成员变量，栈分配速度快且无碎片。
  2. 使用STL容器和智能指针：
    1. std::vector, std::string, std::array：这些STL容器在内部管理一块连续的、动态增长的内存。虽然扩容时需要进行分配和拷贝，但保证了容器内元素的内存是连续的，极大地减少了外部碎片，这是最常用且有效的手段。
    2. 智能指针 (std::unique_ptr, std::shared_ptr,std::shared_weaked)：它们通过RAII机制自动管理内存生命周期，避免了忘记释放导致的“内存泄漏”（一种严重的碎片来源）。但对于防止碎片本身，它们的作用是间接的。
  3. 预分配和资源预留 (Reservation)：
    1. 对于std::vector等容器，如果提前知道大致大小，使用reserve()方法预先分配足够的容量，这避免了多次重新分配、拷贝和释放原内存块，从而减少了碎片的产生。
  4. 使用自定义内存管理：
    1. 使用对象池 (Object Pool)：对于频繁申请释放的、固定大小的小对象（如链表节点、网络数据包），对象池是终极武器。它一次性分配一大块内存（chunk），并在其上构造一个空闲链表，每次分配和归还都是从空闲链表中取放，完全避免了外部碎片，只有极少的内部碎片。
    2. 自定义分配器 (Custom Allocator)：STL容器允许你提供自定义分配器。你可以实现一个基于内存池的分配器，替换默认的new/delete，为特定类型的对象提供高效的、无碎片的分配策略。

什么是内存对齐？

内存对齐是指数据在内存中的存储地址按照特定倍数对齐（通常是数据类型大小的整数倍）。

例如，4字节的int通常存储在4的倍数地址上，8字节的double通常存储在8的倍数地址上。

为什么要内存对齐？

内存对齐的根本原因在于CPU访问数据的效率。现代计算机的硬件设计使得对齐的数据访问速度更快，有些架构甚至要求对齐访问，否则会引发硬件异常。对齐可以减少CPU访问内存的次数，因为某些处理器一次读取内存是一块一块地读取，如果数据跨越了块边界，则需要两次读取操作。

什么是RAII？请举例说明。

  • RAII（Resource Acquisition Is Initialization）是C++的一种管理资源、避免泄漏的惯用法。其核心思想是将资源的生命周期与对象的生命周期绑定，即在构造函数中获取资源，在析构函数中释放资源。这样可以确保资源在对象离开作用域时被正确释放，即使发生异常也是如此。
  • 举例：C++中的std::fstream、std::unique_ptr等。例如，使用std::unique_ptr管理动态分配的内存，当unique_ptr离开作用域时，其析构函数会自动释放所指向的内存。

指针和引用的区别？

  • 指针是一个变量，存储的是另一个变量的地址；引用是一个别名，是另一个变量的同义词。
  • 指针可以被重新赋值指向不同的变量；引用一旦初始化后就不能再改变（始终指向初始化时引用的变量）。
  • 指针可以为空（nullptr）；引用必须绑定到一个已有的对象，不能为空（但可以有悬空引用，这是不安全的）。
  • 指针需要解引用（*）来访问所指向的对象；引用直接使用，无需解引用。
  • 指针有自己独立的内存地址；引用和它引用的对象共享同一个地址（即引用本身的地址就是原变量的地址，但实际上引用在底层通常通过指针实现，但这是编译器细节）。
  • 使用场景：指针常用于需要重新指向、可选参数（可为空）的情况；引用常用于函数参数传递（避免复制）和返回值，特别是用于实现运算符重载和函数链式调用。

野指针与悬空指针产生原因、危害、避免方法是什么？

  • 野指针（Wild Pointer）：未初始化的指针，指向随机的内存地址。
    • 产生原因：定义指针时未初始化。
    • 危害：对野指针解引用可能导致程序崩溃、数据损坏或安全漏洞。
    • 避免方法：总是初始化指针，可以初始化为nullptr。
  • 悬空指针（Dangling Pointer）：指针指向的内存已经被释放，但指针仍然保留着原来的地址。
    • 产生原因：内存被释放后，指针未置空。
    • 危害：访问已释放的内存，导致未定义行为，如数据损坏、程序崩溃。
    • 避免方法：释放内存后立即将指针置为nullptr；使用智能指针替代裸指针。

深拷贝和浅拷贝的区别是什么？

  • 浅拷贝（Shallow Copy）：只拷贝对象的值（包括指针成员变量的值，即地址），而不拷贝指针指向的数据。这样，两个对象的指针成员指向同一块内存。
  • 深拷贝（Deep Copy）：不仅拷贝对象的值，还拷贝指针成员指向的数据，为新对象分配新的内存，并复制内容。这样，两个对象的指针成员指向各自独立的内存块。
    区别在于是否复制指针所指向的数据。深拷贝更安全，但开销更大。

什么是移动语义？它有什么好处？

  • 移动语义是C++11引入的新特性，允许将资源（如动态分配的内存）从一个对象转移到另一个对象，而无需复制。通过右值引用（T&&）和移动构造函数、移动赋值运算符实现。
  • 好处：避免不必要的深拷贝，提高性能，特别是对于管理大量资源的对象（如std::vector、std::string）。移动语义使得临时对象的资源可以被重用，减少了动态内存分配和释放的开销。
