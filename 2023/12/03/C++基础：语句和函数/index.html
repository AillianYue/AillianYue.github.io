C++基础：语句和函数

发表于2023-12-03|更新于2025-05-01|C++C++基础
|总字数:3k|阅读时长:12分钟|浏览量:

VERSION：3.0.0

语句

基础语句类型

类型 说明 示例
表达式语句 由表达式 + 分号构成，执行并丢弃结果 int a = 10;、a++;、func();
复合语句 用{}包裹的多条语句，形成独立作用域 { int x = 5; cout << x; }
声明语句 用于声明变量、函数、类型等实体 double pi = 3.1415;、class MyClass;

选择语句

if 语句

  • 基础结构：if (条件) { 语句 }
  • 扩展形式：if-else、if-else if 嵌套
  • 注意：条件为布尔值，花括号可省略（但不推荐，易出错）

switch 语句

  • 核心：基于整数 / 枚举类型匹配分支，case 后接常量表达式
  • 关键：break 终止当前分支，default 处理未匹配情况

迭代语句

循环类型 执行逻辑 语法示例
while 循环 先判断条件，满足则执行循环体 int i=0; while(i<5) { cout << i++; }
do-while 循环 先执行循环体，再判断条件（至少执行 1 次） int j=0; do { cout << j++; } while(j<5);
传统 for 循环 初始化 → 条件判断 → 循环体 → 更新 for(int k=0; k<5; k++) { cout << k; }
基于范围的 for 循环（C++11+） 遍历数组 / 容器所有元素，简化代码 vector<int> v{1,2,3}; for(auto num : v) { cout << num; }

跳转语句

语句 作用 适用场景
break 终止当前循环 /switch 分支 提前退出循环、结束 switch 匹配
continue 跳过当前循环剩余代码，进入下一次迭代 过滤特定条件的循环迭代
goto 无条件跳转到指定标签位置 仅推荐：多层循环统一退出（慎用，破坏代码结构化）

语句相关辅助特性

结构化绑定（C++17+）

  • 作用：解构聚合类型（数组、结构体、pair 等），一次性绑定多个变量

属性（attribute）

  • 作用：修饰语句 / 声明，告知编译器特殊编译行为（语法：[[属性名]]）
  • 常见属性：[[nodiscard]]（提醒返回值必须使用）、[[deprecated]]（标记废弃）

函数修饰符

修饰符 位置 作用
constexpr 前缀 编译期计算，返回常量（C++11+）
inline 前缀 建议编译器内联展开，减少函数调用开销
static 前缀 限制函数作用域为当前文件，避免重定义
const 后缀（成员函数） 承诺不修改类成员变量
noexcept 后缀 声明函数不会抛出异常（C++11+）

函数重载与匹配

重载规则

  • 核心：函数名相同，参数列表不同（类型 / 个数 / 顺序），返回类型不影响重载

重载解析

  • 原则：编译器优先匹配最佳参数（无隐式转换 > 少量转换 > 大量转换）
  • 示例：add(1, 2.0) 会匹配add(double, double)（int 隐式转 double）

捕获列表

  1. Lambda 捕获的作用域是定义时的词法作用域，仅能捕获定义位置处可见、可访问的变量，和调用位置无关；
  2. 局部变量需显式捕获，全局 / 命名空间变量无需捕获，类成员需捕获this指针才能访问；
  3. 临时表达式不能直接捕获（C++14 初始化捕获除外）。
捕获方式 语法 说明
值捕获 [x] 拷贝外部变量 x，Lambda 内为副本
引用捕获 [&x] 引用外部变量 x，修改会影响原变量
混合捕获 [x, &y] x 值捕获，y 引用捕获
初始化捕获 [z = x + y]
（C++14+）		 捕获时初始化新变量
全部值捕获 [=] 捕获所有外部变量（值拷贝）
全部引用捕获 [&] 捕获所有外部变量（引用）
  • 捕获的是 “定义时” 的作用域，而非 “调用时”
    • Lambda 捕获变量的能力只和它定义的位置有关，和调用的位置无关。哪怕 Lambda 被传递到其他函数、其他作用域调用，它捕获的依然是定义时那个作用域里的变量（注意：引用捕获需关注变量生命周期）。
  • 不同作用域变量的捕获规则，不是所有 “当前能获取的变量” 都需要捕获，分三类情况：
    1. 局部作用域变量（函数内、代码块{}内的变量）：必须显式捕获（值[x]/ 引用[&x]/ 全部[=]/[&]）才能在 Lambda 内访问；
    2. 全局 / 命名空间作用域变量：无需捕获，Lambda 内可直接访问（因为全局作用域全程可见）；
    3. 类成员变量：Lambda 定义在类成员函数内时，不能直接捕获成员变量，需捕获this指针（[this]），通过this->成员名访问（本质是捕获当前对象的指针）。
  • 不能直接捕获的变量，即使在当前作用域能获取，以下情况也不能直接捕获：
    • 临时对象 / 表达式结果（比如[x+1]）：C++11 不支持，C++14 可通过初始化捕获解决（[val = x+1]）；
    • 寄存器变量（register int x）：C++17 后寄存器关键字弱化，本质还是局部变量，可正常捕获；
    • 函数参数包（可变参数模板里的Args…）：需结合参数包展开处理。
