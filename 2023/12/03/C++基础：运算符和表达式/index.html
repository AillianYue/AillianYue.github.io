C++基础：运算符和表达式

发表于2023-12-03|更新于2025-05-01|C++C++基础
|总字数:5.3k|阅读时长:20分钟|浏览量:

VERSION：3.0.0

核心概念

定义

  • 运算符：用于对操作数执行特定运算的符号（如+、&&、->），分为单目（1个操作数）、双目（2个操作数）、三目（3个操作数，仅?:）。
  • 表达式：由运算符和操作数组成的合法代码片段，最终会产生一个值（如a + b * 2、x > 0 && y != 0）。
  • 左值（lvalue）：可被修改、能取地址的实体（变量、对象成员等），如int a、p->name。
  • 右值（rvalue）：临时值、字面量或不可修改的表达式结果，如3、a + b、std::move(x)。

核心特性

  • 优先级：决定运算符的执行先后顺序（如乘除优先于加减）。
  • 结合性：同优先级运算符连续出现时的执行方向（左结合/右结合）。
  • 求值顺序：操作数的计算顺序（C++17后部分场景明确，需重点关注）。

运算符详解

逻辑运算符（短路求值特性）

连接布尔表达式，执行逻辑判断，结果为bool类型，支持短路求值（避免不必要计算，提升效率）。

运算符 功能描述 短路规则 注意事项
&& 逻辑与：两侧均为true则结果为true 左侧为false时，右侧不再求值 避免在右侧写有副作用的代码（如a++）
` ` 逻辑或：任一为true则结果为true
! 逻辑非：单目运算符，取反 无短路（仅一个操作数） 避免对非布尔值直接使用（如!3，虽合法但可读性差）

算术运算符（含位运算符）

基础算术运算符：对数值类型执行运算，结果类型可能发生隐式转换，需注意溢出和整数除法问题。

运算符 功能 示例 关键注意事项
+ 加法/正号 a + b、+3 单目+无实际作用，仅用于语法兼容
- 减法/负号 a - b、-x 单目-对有符号类型取负，需避免溢出（如-INT_MIN未定义）
* 乘法 m * n 整数乘法易溢出，大数值建议用long long或std::mul_overflow（C++20）
/ 除法 5 / 2 = 2、5.0 / 2 = 2.5 整数除法舍弃小数，负数除法结果符号与左操作数一致（-5/2=-2）
% 取余（模运算） 7%3=1、-7%3=-1 操作数必须为整数，结果符号与左操作数一致

位运算符（补充原笔记缺失）：直接操作二进制位，常用于底层编程、性能优化场景。

运算符 功能 示例 应用场景
& 按位与 0b1010 & 0b0110 = 0b0010 位掩码（提取特定位）、判断奇偶（x & 1）
` ` 按位或 `0b1010
^ 按位异或 0b1010 ^ 0b0110 = 0b1100 交换变量（无需临时变量）、翻转位
~ 按位取反 ~0b1010 = 0b0101（无符号） 注意符号位扩展（有符号类型会变成负数）
<< 左移 x << 1（等价于x*2，无溢出时） 快速乘法、位运算优化
>> 右移 x >> 1（等价于x/2，正数） 有符号类型右移可能算术移位（补符号位）或逻辑移位（补0），依赖编译器

赋值运算符（基础+复合）

基础赋值（=）

  • 功能：将右侧表达式的值赋给左侧左值，返回左值引用，支持链式赋值。
  • 示例：
  • 注意：初始化与赋值的区别（初始化是创建变量时赋值，赋值是修改已有变量的值）。

复合赋值运算符

将“运算+赋值”合并，左操作数仅求值一次（效率高于普通写法），优先级与=一致。

运算符 等价写法 示例
+= a = a + b a += 3（a自增3）
-= a = a - b a -= b
*= a = a * b x *= 2（x翻倍）
/= a = a / b y /= 3
%= a = a % b z %= 5

自增/自减运算符（前置vs后置）

形式 操作时机 返回值类型 示例 结果 适用场景
前置++ 先自增，再返回 左值（变量本身） int a=2; int b=++a; a=3，b=3 单独自增、需立即使用自增后的值
后置++ 先返回，再自增 右值（自增前的副本） int a=2; int b=a++; a=3，b=2 循环迭代（for(int i=0; i<10; i++)）、无需立即使用原值
  • 性能：前置++效率高于后置++（后置需创建临时副本），尤其对于自定义类型（如std::vector::iterator），优先使用前置。
  • 陷阱：避免在同一表达式中多次修改同一变量（如a = a++ + ++a），会导致未定义行为。

比较运算符（含浮点数比较技巧）

比较两个操作数的关系，结果为bool类型，常用于条件判断和循环控制。

运算符 功能 示例
== 相等 x == y
!= 不相等 x != 0
>/< 大于/小于 a > b、c < 10
>=/<= 大于等于/小于等于 age >= 18

浮点数比较陷阱：因精度误差，不能直接用==/!=比较浮点数，需判断差值是否小于极小值（如1e-6）。

自定义类型比较：需重载==/!=等运算符，实现深比较（避免浅比较导致的逻辑错误）。

成员访问运算符（. 与 ->）

功能：访问类/结构体/联合体的成员

运算符 适用场景 示例
. 直接访问对象成员 Person p; p.name = "Alice";
-> 通过指针访问对象成员 Person* p = &p1; p->age = 25;

底层逻辑：p->member 等价于 (*p).member，编译器会自动优化，两者效率一致。

三元条件运算符（?:）

语法：条件表达式 ? 表达式1 : 表达式2（C++中唯一的三目运算符）

核心规则：

  • 条件为true时，执行并返回表达式1的值；否则返回表达式2的值。
  • 表达式1和表达式2类型需兼容（可隐式转换为同一类型）。
  • 结合性：右结合（支持嵌套，但不推荐超过2层）。

逗号运算符（,）

语法：表达式1, 表达式2, ..., 表达式n

核心规则：

  • 优先级最低（低于赋值运算符），结合性左到右。
  • 依次执行所有表达式，最终返回最后一个表达式的值。
  • 用途：在单个语句中执行多个操作（如for循环初始化/更新）。

高级特性：运算符重载

重载规则与限制

核心目的：让自定义类型（类/结构体）能像内置类型一样使用运算符，提升代码可读性。

不可重载的运算符（必记）：

  • .（成员访问）
  • .*（成员指针访问）
  • ::（作用域解析）
  • ? :（三元条件）
  • sizeof（大小计算）
  • typeid（类型识别）
  • const_cast/dynamic_cast等转换运算符。

关键规则

  • 重载后优先级、结合性、操作数个数不变。
  • 必须与自定义类型相关（不能重载内置类型的运算符，如int operator+(int a, int b)）。
  • 不能新增运算符（只能重载已有运算符）。

类型转换机制

隐式类型转换

定义：编译器自动完成的转换，无需程序员干预，分为标准转换和用户自定义转换。

标准转换（内置类型）

转换类型 示例 安全性
算术转换 int a = 10; double b = a;（小→大） 安全（无数据丢失）
算术转换 double a = 3.14; int b = a;（大→小） 不安全（数据丢失）
布尔转换 if (3) { ... }（非0→true） 安全（语法合法）
指针转换 Derived* d = new Derived; Base* b = d;（向上转型） 安全
指针转换 void* p = &a;（任意指针→void*） 安全

显式类型转换

C++转换（推荐）分为4种，功能明确、易定位、安全性更高。

转换类型 语法 核心用途 安全性 示例
static_cast static_cast<目标类型>(源) 1. 内置类型显式转换；2. 类层次向上转型；3. 空指针→任意指针；4. 编译期确定的转换 中等（无运行时检查） int num = static_cast<int>(3.14);
dynamic_cast dynamic_cast<目标类型>(源) 类层次向下转型（基类→派生类）或交叉转型 高（运行时检查类型兼容性） Derived* d = dynamic_cast<Derived*>(base_ptr);
const_cast const_cast<目标类型>(源) 仅移除/添加const/volatile修饰符 低（修改原const变量未定义） int* p = const_cast<int*>(&const_var);
reinterpret_cast reinterpret_cast<目标类型>(源) 底层二进制位重新解释（不相关类型转换） 极低（平台相关，易未定义） uintptr_t addr = reinterpret_cast<uintptr_t>(&x);
  1. dynamic_cast：
- 仅支持有虚函数的类（需运行时类型信息RTTI）。
- 指针转换失败返回`nullptr`，引用转换失败抛出`std::bad_cast`异常。
  1. const_cast：
- 仅修改`const`属性，不能改变类型。
- 若原变量是`const`（如`const int a=10`），通过`const_cast`修改会导致未定义行为。
  1. reinterpret_cast：
    • 用途：指针→整数、整数→指针、不同类型指针转换（如int*→char*）。
    • 注意：转换结果依赖平台，移植性差，仅在底层编程中使用。

C 风格转换（不推荐）：

  • 语法：(目标类型)源数据（如(int)3.14）。
  • 缺陷：功能单一、无安全检查、不易定位（大型项目中难以搜索）。
  • 建议：C++开发中完全替换为新式转换。

转换安全原则

  1. 避免大类型→小类型的隐式转换（如double→int）。
  2. 自定义类型转换优先用explicit修饰（C++11+），强制显式转换。

关键修饰符与表达式特性

常量表达式（constexpr）

定义：编译期就能确定值的表达式，用constexpr关键字标识，可用于编译期常量场景（如数组大小、模板参数），提升程序效率（避免运行时计算）。

  • constexpr变量：必须用常量表达式初始化，值在编译期确定（区别于const：const可能是运行时常量）。
  • constexpr函数：
- C++11：函数体仅1条`return`语句，无副作用。
- C++14+：支持循环、条件判断等复杂逻辑。
- 特性：参数为常量表达式时，函数在编译期执行；否则在运行时执行（兼容两种场景）。
  • constexpr类与构造函数：C++11+支持constexpr构造函数，使类对象能在编译期初始化。

易变表达式（volatile）

定义：volatile是类型修饰符，告诉编译器：“该变量的值可能被程序外部因素修改”（如硬件中断、多线程共享变量），禁止编译器对该变量进行优化（如缓存到寄存器）。

  1. 硬件寄存器访问：硬件设备可能主动修改寄存器值，需每次从内存读取最新值。
  1. 多线程共享变量：volatile确保线程读取到内存中的最新值，但不保证原子性（需配合std::atomic或互斥锁）。

常见误区

  • volatile = 线程安全：错误！volatile仅禁止优化，不保证原子性和内存序。
  • volatile可替代const：错误！需同时禁止优化和只读时，用volatile const。
  • 对自定义类型使用volatile：需重载volatile版本的成员函数，否则无法通过volatile对象访问。

运算符优先级与结合性

优先级 运算符类别 代表运算符 结合性 示例
1（最高） 括号/成员访问 ()、[]、.、-> 左结合 p->func()、(a + b) * c
2 单目运算符 ++（前置）、--（前置）、!、-（负号）、*（解引用）、&（取地址） 右结合 !a、++i、*ptr
3 算术运算符（乘除/位运算） *、/、%、<<、>>、& 左结合 a * b / c、x << 1
4 算术运算符（加减/位运算） +、-、` 、^` 左结合
5 比较运算符 >、<、>=、<= 左结合 a > b && c < d
6 相等性判断 ==、!= 左结合 `a == b
7 逻辑与 && 左结合 cond1 && cond2 && cond3
8 逻辑或 ` `
9 三元条件运算符 ? : 右结合 a > b ? a : b
10（最低） 赋值运算符（基础+复合） =、+=、-=、*=、/=等 右结合 a = b += 3

求值顺序与未定义行为

核心概念：

  • 优先级：决定运算符的执行顺序（如a + b * c先算b*c）。
  • 求值顺序：决定操作数的计算顺序（如f(a) + g(b)中，f(a)和g(b)的调用顺序）。

C++17前后的关键变化：

场景 C++17前 C++17后
a + b a和b的求值顺序未定义 仍未定义
f(a) + g(b) f(a)和g(b)的调用顺序未定义 仍未定义
a = b + c b和c的求值顺序未定义 仍未定义
new T(a) a和new T的顺序未定义 先求值a，再分配内存
a << b << c 未定义 左到右（先a<<b，再结果<<c）
std::cout << f() << g() 未定义 左到右（先f()，再g()）

未定义行为（UB）的常见场景：

未定义行为：编译器可任意处理，可能崩溃、结果错误或看似正常，需严格避免。

  1. 同一表达式中多次修改同一变量：a = a++ + ++a、i++ && i++。
  2. 访问未初始化的变量：int x; cout << x;。
  3. 解引用空指针或野指针：int* p = nullptr; *p = 10;。
  4. 整数溢出：int x = INT_MAX; x++;。
  5. 修改const变量（通过const_cast）：const int a=10; int* p=&a; *p=20;。

注意事项

运算符使用

  1. 优先使用前置++/--：尤其是自定义类型，避免后置的临时副本开销。
  2. 浮点数比较用差值法：避免直接用==/!=，使用std::fabs(a - b) < eps。
  3. 避免滥用逗号运算符：仅在for循环等必要场景使用，其他场景拆分语句提升可读性。
  4. 三元条件运算符不嵌套过深：超过2层嵌套时，改用if-else。
  5. 赋值表达式用于条件判断时加括号：if ((ret = func()) != 0)，避免优先级问题。

运算符重载

  1. 重载运算符需符合直觉：如+应表示“加法”，而非其他逻辑（如减法）。
  2. 支持交换律：双目运算符（如+）用非成员函数重载，确保a + b和b + a均可用。
  3. 重载==时同步重载!=：遵循“相等”与“不等”的逻辑一致性。
  4. 重载赋值运算符时注意深拷贝：避免浅拷贝导致的双重释放。

类型转换

  1. 优先使用显式转换：自定义转换构造函数和类型转换运算符加explicit。
  2. 避免reinterpret_cast：仅在底层编程（如硬件访问）中使用，且需注释说明。
  3. dynamic_cast配合空指针检查：指针转型后先判断是否为nullptr，避免解引用失败。
  4. const_cast仅用于移除临时const：不修改原const变量。

现代C++特性应用

  1. 用constexpr优化编译期计算：如常量定义、简单函数，提升运行效率。
  2. 用std::atomic替代volatile：多线程共享变量需原子性时，优先使用std::atomic。
  3. 用新式转换替代C风格转换：提升代码可读性和安全性。
  4. 重载operator<<支持自定义类型输出：方便调试和日志打印。
