整数字面量规则

字面量是直接写在代码中的整数，需通过前缀指定进制、后缀指定类型，支持单引号分隔提升可读性：

  • 进制前缀：
    • 二进制：0b（例：0b1010 → 十进制 10，C++14 及以后支持）
    • 八进制：0（例：012 → 十进制 10）
    • 十进制：默认（例：10）
    • 十六进制：0x（例：0xA → 十进制 10，字母大小写均可）
  • 类型后缀：
    • unsigned：u 或 U（例：100u → unsigned int）
    • long：l 或 L（例：100L → long）
    • long long：ll 或 LL（例：100LL → long long）

明确大小的整数类型（库）

上述类型（如 long）的字节数随平台变化，若需固定位数的整数（如 8 位、16 位），需包含头文件 <cstdint>，使用以下类型：

  • 有符号：int8_t（8 位）、int16_t（16 位）、int32_t（32 位）、int64_t（64 位）
  • 无符号：uint8_t、uint16_t、uint32_t、uint64_t

浮点类型

float 有 4 4 4 4 %g
double 有 8 8 8 8 %lg
long double 有 8 8 8 16 %Lg
  • float：单精度
  • double：双精度
  • long double：扩展精度

浮点字面量规则

  • 类型后缀：
    • float：f 或 F（例：3.14f → 否则默认是 double）
    • long double：l 或 L（例：3.14L）
    • double：默认（例：3.14）
  • 科学计数法：支持 e 或 E 表示指数（例：1.23e4 → 12300，1.23e-4 → 0.000123）

字符类型

用于存储单个字符（如 ‘a’、’1’），分为窄字符（1 字节）和宽字符（2/4 字节，支持 Unicode）。

窄字符

（1 字节）

类型 特性说明 用途
char 默认字符类型，可能有符号也可能无符号（取决于编译器） 存储 ASCII 字符（如 ‘a’）
signed char 明确为有符号（范围：-128 ~ 127） 存储带符号的 8 位数据
unsigned char 明确为无符号（范围：0 ~ 255） 存储无符号的 8 位数据

宽字符

（2/4 字节，支持 Unicode）

类型 字节数 对应字符集 用途
char16_t 2 UTF-16 存储 2 字节 Unicode 字符
char32_t 4 UTF-32 存储 4 字节 Unicode 字符
wchar_t 2（Windows）/4（Linux/Mac） 平台依赖的宽字符集 适配平台最大字符（如 Unicode）

字符字面量规则

字符字面量需用单引号（''）包裹，通过前缀指定字符类型：

字符类型 前缀 示例
char 无 'a'、'1'
wchar_t L L'a'、L'中'
char16_t u u'a'、u'\u0041'
char32_t U U'a'、U'\U0001F37A'

转义序列

用于表示 “不可见字符” 或 “特殊字符”（如换行、单引号），常见转义序列：

  • \n：换行
  • \t：制表符（Tab）
  • \'：单引号（避免与字面量分隔符冲突）
  • \"：双引号（用于字符串中）
  • \\：反斜杠本身

Unicode 转义字符

（通用字符名）

通过 “前缀 + 码位” 表示 Unicode 字符，支持两种格式：

  1. \u + 4 位十六进制码位（例：u'\u0041' → 对应字符 ‘A’）
  2. \U + 8 位十六进制码位（例：U'\U0001F37A' → 对应啤酒杯字符🍺）

ASCII

美国信息交换标准代码(American Standard Code for Information Interchange，ASCII)表

ASCII 码是 “字符与整数的映射表”，仅覆盖英文字符，常用对应关系：

十进制值 字符 说明
32 (空格) 空格字符
48 0 数字 0
65 A 大写字母 A
97 a 小写字母 a
127 DEL 删除字符

布尔类型

用途：表示 “真” 或 “假”，仅两个取值：true（本质是 1）、false（本质是 0）。

占用字节：通常 1 字节（编译器可能优化，但至少 1 位）。

注意：整数可隐式转换为 bool（0→false，非 0→true），反之 bool 转换为整数（false→0，true→1）。

std::byte类型

  • 头文件：需包含 <cstddef>。
  • 特性：仅用于存储原始字节数据，支持按位运算（&、|、^、~），但不支持算术运算（如 +、-，避免误操作）。
  • 优势：比char或int更安全 —— 用std::byte处理二进制数据时，编译器会禁止无效运算（如byte + 1报错），减少调试错误。

size_t类型

  • 头文件：需包含 <cstddef>（C++ 中位于std命名空间，可写std::size_t）。
  • 用途：表示 “对象大小” 或 “索引值”（如数组索引、sizeof返回值），是无符号类型。
  • 大小：平台依赖，保证能存储 “当前系统最大对象的字节数”——64 位系统通常等同于unsigned long long（8 字节），32 位系统通常等同于unsigned int（4 字节）。
  • 格式符：printf输出时，用%zd（十进制）或%zx（十六进制）。
  • 与sizeof的关系：sizeof(类型/变量)返回值的类型是size_t

数组

数组是 “相同类型数据的连续存储”，C 风格字符串是 “以\0（null 字符）结尾的 char 数组”。

C 风格字符串

  • 本质：char数组，必须在末尾附加\0（值为 0）表示字符串结束，否则会导致 “越界访问”。
  • 示例：char str[] = "hello"; → 实际存储为 h e l l o \0（共 6 个字节，含\0）。

字符串字面量

  • 定义：用双引号（""）包裹的文本，支持 Unicode 前缀（与宽字符对应）。
  • 注意：Unicode 字符串打印需用 <cwchar> 头文件的wprintf（直接用printf可能乱码）。

数组初始化

数组初始化支持 “长度指定” 或 “长度推断”，未显式初始化的元素默认置 0（仅大括号初始化时）。

初始化方式 示例 结果（元素值） 说明
长度推断 + 全初始化 int arr1[]{1, 2, 3}; [1, 2, 3] 长度由初始化元素个数决定
指定长度 + 零初始化 int arr2[5]{}; [0, 0, 0, 0, 0] 大括号空表示所有元素置 0
指定长度 + 部分初始化 int arr3[5]{1, 2, 3}; [1, 2, 3, 0, 0] 未显式初始化的元素置 0
仅指定长度（无初始化） int arr4[5]; 随机值（未定义） 栈上数组未初始化，值不确定

用户自定义类型

枚举类型

enum比enum class使用起来更不安全，如无必要，请不要使用enum，请使用enum class。C++支持enum主要是出于历史原因，特别是基于与C代码的互操作。

普通数据（POD）类

POD（Plain-Old-Data）即 “简单数据容器”，用struct关键字定义，仅包含数据成员（无成员函数、继承等），本质是 “C 风格结构体”。

特性：内存布局连续，可直接按字节拷贝（如<font style="color:rgb(0, 0, 0);">memcpy</font>），初始化规则与基本类型类似。

大括号初始化

  • 优势：无论对象的作用域或类型如何，大括号初始化总是适用，其他方法则不总适用
  • 背景：解决C++初始化语法混乱问题，平滑过渡不同对象生命周期的初始化需求

初始化基本类型为0

基本类型（int、float 等）的初始化分为 “初始化为 0” 和 “初始化为特定值” 两类。

方式 示例 说明
等号赋值 <font style="color:rgb(0, 0, 0);">int a = 0;</font> 传统方式，直观
大括号初始化 <font style="color:rgb(0, 0, 0);">int a{};</font> 现代方式，无窄化转换风险
等号 + 大括号初始化 <font style="color:rgb(0, 0, 0);">int a = {};</font> 兼容传统习惯，效果同<font style="color:rgb(0, 0, 0);">a{}</font>

初始化基本类型为某值

方式 示例 说明
等号赋值 int b = 6; 传统方式
大括号初始化 int b{6}; 现代推荐，防止窄化（如int{3.14}报错）
等号 + 大括号初始化 int b = {6}; 兼容传统，效果同b{6}
圆括号初始化 int b(6); 类构造函数风格，基本类型也支持

初始化POD

初始化POD的语法大多遵循基本类型的初始化语法。

将POD对象初始化为零与将基本类型的对象初始化为零类似。大括号初始化和等号加大括号的初始化产生相同的代码：字段初始化为零。

不能对POD使用“= 0”的初始化方法

printf

printf 是 C 语言标准库中的一个函数，来自于 <stdio.h> 头文件，虽然 C++ 更倾向于使用 cout，但 printf 在某些情况下仍然非常有用。

printf格式修饰符

格式说明符可以包含额外的标志+宽度+精度来控制输出格式：

  • 标志：- 表示左对齐，+ 在正数前加加号。
  • 宽度：指定输出字段的最小宽度。
  • 精度：用于浮点数和字符串的显示精度。

举例：%-8.3e表示打印一个浮点数，左对齐，占位宽度为8，有效数字为3
