ROS 创建工作空间与功能包

发表于2021-07-16|更新于2021-07-16|历史文档ROS基础
|总字数:423|阅读时长:1分钟|浏览量:

工作空间（workspace）是一个存放工程开发相关文件的文件夹
有四个重要的文件夹

  • src：代码空间（Source Space）
  • build：编译空间（Build Space）
  • devel：开发空间（Development Space）
  • install：安装空间（Install Space）

catkin编译系统下的工作空间如下图所示

创建工作空间

mkdir -p ~/catkin_ws/src，其中，-p 确保目录名称存在，不存在的就建一个。~代表当前用户根目录
cd /catkin_ws/src
**catkin_init_workspace**

编译工作空间

cd ~/catkin_ws/
catkin_make

换句话说就是新建文件夹并命名为catkin_ws或者自己喜欢的名字，双击进入/catkin_ws，然后新建文件夹并命名为src，双击进入/src，运行catkin_init_workspace，此时/src文件夹会出现 CMakeLists.txt 文件，如下图
，然后返回工作空间/catkin_ws，运行catkin_make，此时多出了，/devel，/build，如图

再运行catkin_make install，多出了/install文件夹，如图

工作空间就创建好了。

设置环境变量

如果要运行功能包中的某个程序，需要先设置工作空间的环境变量
source** devel/setup.bash

创建功能包

catkin_create_pkg <package_name> [depend1] [depend2]
功能包在/src下创建，比如创建名为test_pkg的功能包，此功能包依赖std_msgs、rospy、roscpp，则运行
catkin_create_pkg test_pkg std_msgs rospy roscpp，则/src下会出现一个名为test_pkg的文件夹，如图

/test_pkg其内容为下图，其中CMakeList.txt和package.xml是必定存在的

  • include：包含的头文件
  • src：源码
  • CMakeList.txt
    • 描述编译规则
  • package.xml
    • 开头涉及名字，版本号，描述信息，email，开源许可证等信息
    • 后面是依赖信息
