3、安装

首先更新你的包：运行这一行代码

sudo apt update

完成后如下图


然后安装noetic（desktop-full代表推荐配置，不是真的full，某些包还是需要手动安装)，运行这一行代码

sudo apt install ros-noetic-desktop-full


输入**y**继续，等它下载一会儿，然后报错了（我用的阿里云的镜像）


根据终端返回的建议用

sudo apt-get update


好了之后重新运行sudo apt install ros-noetic-desktop-full，就没问题了


再重新运行一下sudo apt install ros-noetic-desktop-full确认一下是最新版本

4、初始化rosdep

初始化rosdep，运行这一行代码

sudo rosdep init

如果报错sudo: rosdep: command not found，说明你的系统没有装rosdep，则需要运行这一行代码来先安装rosdep
sudo apt install python3-rosdep


如果报错
ERROR: cannot download default sources list from:https://raw.githubusercontent.com/ros/rosdistro/master/rosdep/sources.list.d/20-default.list Website may be down.
则运行这一行代码sudo gedit /etc/hosts
然后在打开的hosts文件中添加如下的内容199.232.28.133 raw.githubusercontent.com，然后保存退出，199.232.28.133这个数字不一定是对的，需要查找
添加到图示的地方还是添加到末尾我也不确定，都可以

然后重新运行sudo rosdep init


出现上面这个画面就说明完成了sudo rosdep init了
如果报错ERROR: default sources list file already exists: /etc/ros /rosdep/sources.list.d/20-default.listPlease delete if you wish to re-initialize
说明这个初始化文件已经存在了
20-default.list 它长这样

可以将其删掉，用sudo rm /etc/ros/rosdep/sources.list.d/20-default.list
然后重新sudo rosdep init即可，不过这个初始化完成了没必要删掉重新初始化，所以这个错误可以忽略
继续下个步骤，更新rosdep，运行这一行代码

rosdep update

必报错
不出所料，报错（unable to process source或者error loading sources list: The read operation timed out），这一步报错都是网络问题，


参考这篇博客：略微有点小麻烦，但是亲测有效

5、环境设置

每次启动命令行都会把ROS环境变量自动添加到了bash会话中，运行这一行代码
echo "source /opt/ros/noetic/setup.bash" >> ~/.bashrc
然后运行这一行代码
source ~/.bashrc
然后运行这一行代码试试效果，成功的话可以看到一些ros的命令
ros+tab+tab

然后我们启动一下小海龟的仿真器试一试，按ctrl+alt+t打开一个新的终端，输入以下命令
rosrun turtlesim turtlesim_node
按ctrl+alt+t再打开一个新的终端，输入以下命令
rosrun turtlesim turtle_teleop_key
这时候你按键盘上的↑↓←→键就可以控制小海龟移动了！！！
