ClaudeCode基础07：常用命令

发表于2026-06-16|更新于2026-06-26|VibeCodingClaudeCode
|总字数:931|阅读时长:3分钟|浏览量:

VERSION：1.0.0

会话内命令速查

会话管理

命令 作用
/clear 硬重置，清空所有对话历史，开始全新会话
/compact [指令] 压缩早期对话释放上下文空间，可指定保留重点
/rewind 回退到之前检查点（可分别回退对话、代码或两者）
/btw <问题> 问一个不干扰主线的问题，答案不入对话历史
/rename <名称> 重命名当前会话，便于后续恢复
/export [文件名] 导出对话为文档
/branch <名称> 分支当前会话尝试替代方案（原名 /fork）

模式切换

命令 作用
/plan 进入只读规划模式，允许探索和讨论，不做任何更改
/fast 切换快速模式（使用 Opus 但更快输出）
/effort <级别> 设置推理深度：low | medium | high | xhigh | max
/model <模型> 切换模型（如 claude-sonnet-4-6、claude-opus-4-8）

查看与诊断

命令 作用
/context 以彩色网格可视化当前上下文使用情况
/cost 查看 token 用量、配额和花费
/diff 交互式查看当前会话的所有文件变更
/doctor 诊断安装状态：Node.js 版本、API 连通性、认证有效性
/status 显示版本号、当前模型、账户信息

项目配置

命令 作用
/init 根据项目结构自动生成 CLAUDE.md
/memory 在会话中直接编辑 CLAUDE.md
/config 打开设置面板
/permissions 管理工具和 skill 的权限规则
/add-dir <路径> 添加额外工作目录（monorepo 利器）
/mcp 管理 MCP 服务器连接和 OAuth
/plugin 浏览和管理插件市场

代码工作流（内置 skills）

命令 作用
/code-review [effort] 审查当前 diff 的代码正确性和质量问题
/security-review 审查当前分支的安全漏洞
/verify 运行应用并观察行为，确认改动生效
/run 根据项目类型自动检测并启动应用
/test 运行测试（自动检测测试框架和命令）

高级自动化（内置 skills）

命令 作用
/batch <指令> 使用 worktree 隔离，大规模并行修改多个文件
/loop <间隔> <指令> 按固定间隔重复执行指定任务
/agents 创建和管理 sub-agent 配置
/ultrareview 云端多智能体深度代码审查
/ultraplan 云端深度规划分析

辅助

命令 作用
/help 显示帮助信息
/release-notes 查看当前版本的更新日志
/exit 退出 Claude Code（别名 /quit）

CLI 命令速查

启动与会话

命令 作用
claude 启动交互式会话
claude "prompt" 带初始提示启动交互式会话
claude -p "query" 非交互模式，打印结果后退出
cat file | claude -p "query" 管道输入文件内容
claude -c 继续当前目录最近一次的会话
claude -r <name> 恢复指定名称或 ID 的会话

维护与管理

命令 作用
claude update 更新到最新版本
claude mcp 配置 MCP 服务器

常用 Flags

Flag 作用
--model 指定模型（如 --model claude-sonnet-4-6）
--effort 设置推理深度
--allowed-tools 允许白名单工具免确认执行
--dangerously-skip-permissions 跳过所有权限确认
--add-dir 添加额外工作目录
--worktree / -w 在隔离的 git worktree 中启动
--system-prompt 替换默认系统提示词

快速参考

操作 作用
Esc + Esc 打开回退菜单（rewind）
Ctrl + T 任务列表（todos）
输入 / 查看所有可用命令列表
claude --help 查看 CLI 帮助（部分 flag 不在此列出，以官方文档为准）

需要安装的 community skills（如 grill-me、caveman、handoff 等）请见上篇：/2026/06/15/ClaudeCode基础06：skills推荐
