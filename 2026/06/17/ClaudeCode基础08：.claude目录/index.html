ClaudeCode基础08：.claude目录

发表于2026-06-17|更新于2026-06-26|VibeCodingClaudeCode
|总字数:1.9k|阅读时长:7分钟|浏览量:

VERSION：1.0.4

.claude/ 目录是 Claude Code 的配置中枢。无论是项目上下文（CLAUDE.md）、可复用技能（skills）、自动化钩子（hooks）、权限与模型设置（settings.json），还是子代理定义（agents）、工作流脚本（workflows），都组织在这个目录下。

本文从项目级的 .claude/ 目录出发，绘制全景图，逐一速览各组件的职责与用法。

双层架构

Claude Code 的配置存在于两个层级：

层级 路径 提交到 Git 用途
全局（个人） ~/.claude/ ❌ 个人偏好、全局技能、全局模型设置
项目（共享） .claude/（仓库根目录） ✅ 项目说明、团队共享技能、项目级配置
  • 全局配置适用于你所有的 Claude 会话——你个人的编码习惯、全局技能等。
  • 项目配置提交到 Git，随仓库分享给所有协作者——确保每个人在同一个 AI 工作环境下协作。

两层的同名配置项会合并（数组相加，标量优先级在项目 > 全局）。

本系列之前的文章主要围绕的是项目级配置。本文基于项目级 .claude/ 目录，差别不大

各组件简介

CLAUDE.md

项目指令文件，每个新会话默认注入上下文中。这是配置 Claude Code 投入产出比最高的文件。

作用是快速向 AI 交代项目背景：技术栈、构建命令、目录结构、代码规范、常见陷阱等。建议保持在 200-300 行以内。

settings.json

项目配置的枢纽，控制 Claude 的行为方式。

主要区块：

permissions（权限控制）

  • allow — 白名单命令，运行时不弹权限确认
  • deny — 阻止特定命令执行

可以使用 /permissions 命令交互式管理或直接编辑此文件。

hooks（钩子）

  • PreToolUse — 工具调用前触发（如提交前检查）
  • PostToolUse — 工具调用后触发（如自动格式化）

钩子的实际脚本放在 .claude/hooks/ 目录中，settings.json 只负责配置触发时机和匹配规则。

model（模型选择）

指定项目默认使用的 Claude 模型。也可以在 CLI 中通过 --model 临时覆盖。

env（环境变量）

设置运行命令时注入的环境变量，避免在 shell 配置中重复声明。

完整示例

settings.local.json

与 settings.json 结构相同，但不会提交到 Git（自动被 .gitignore 排除）。适合存放个人偏好——比如你的个人权限白名单、偏好的模型、本地专用环境变量等。运行时与 settings.json 合并，优先级更高。

CLAUDE.local.md

与 CLAUDE.md 结构相同，但不会提交到 Git（需要手动在 .gitignore 中添加）。适合记录个人备忘、本地开发环境特有的信息、不想让团队看到的实验性配置等。

注意：此文件不像 settings.local.json 那样被自动忽略，需要手动添加到 .gitignore。

.mcp.json

配置 MCP（Model Context Protocol）服务器。MCP 是 Claude Code 获取外部数据或工具的协议——通过配置 MCP 服务器，Claude 可以查询数据库、读取文件系统、调用第三方 API 等。

注意：.mcp.json 只在项目级生效，全局级不支持。

skills/

技能是一种结构化的可复用 prompt，通过 /<name> 调用或配置自动触发。用于将重复性工作流（代码审查、部署、格式化等）封装成可复用的命令。

在 .claude/skills/ 下每个技能一个子目录，目录名即命令名。

输入 /review 即可触发。

hooks/

存放钩子可执行脚本，供 settings.json 中配置的钩子引用。钩子可以是 shell 脚本（.sh）、JavaScript（.js）或任何可执行文件。

rules/

当 CLAUDE.md 变得臃肿时，可以将特定主题的指令拆分到 rules/ 目录下。每个 .md 文件是独立的规则文件，可配置路径匹配，只作用于指定路径下的对话。

agents/

定义可复用的子代理，每个代理拥有独立的 prompt 和工具集。适合将特定任务（如数据库专家、UI 设计师、安全审计员）封装为独立代理，在主会话中按需调用。

workflows/

动态多代理编排脚本，用 JavaScript 编写，可并行或串行调用多个子代理协作完成复杂任务。适合代码迁移、大规模审查、多维度分析等场景。

output-styles/

自定义系统提示的特定段落，用来控制 Claude 回复的语气、格式或详细程度。比如定义"简洁模式"、"详细模式"等风格配置。

agent-memory/

子代理的持久化状态存储。当子代理需要跨对话记住信息时（如用户偏好、已完成的任务状态），会自动读写此目录下的内容。
