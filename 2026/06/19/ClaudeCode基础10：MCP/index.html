ClaudeCode基础10：MCP

MCP 是什么

MCP（Model Context Protocol，模型上下文协议） 是 Anthropic 推出的开放协议，旨在标准化 LLM 应用程序与外部工具/数据源之间的通信方式。

可以把它想象成 AI 世界的 USB-C——以前每个工具都要独立适配，现在通过统一接口即插即用。

三层架构

MCP 采用清晰的 Host-Client-Server 三层结构：

层 角色 谁负责
Host 发起请求的 AI 应用 Claude Code / Claude Desktop
Client 协议实现，负责与 Server 通信 Host 内置，无需我们操心
Server 暴露工具（tools）、资源（resources）、提示词（prompts） 我们配置或编写

在 Claude Code 的语境下：Host 和 Client 都已经就绪，你只需要配置 Server 即可。Claude Code 会自动发现 .mcp.json 中定义的 Server，把它的工具注入到 AI 可调用的工具集中。

为什么需要 MCP

在 MCP 出现之前，给 LLM 添加外部工具能力通常需要：

  1. 写 function calling 适配代码
  2. 手动注册每个 API
  3. 维护不同模型的不兼容接口

MCP 解决了这些问题：

  • 统一接口：无论背后是什么服务，都通过标准化的 tool/resource 暴露
  • 动态发现：Server 启动时告诉 Client 它有什么工具，无需硬编码
  • 解耦：工具升级只需更新 Server，Host 端零改动
  • 生态共享：同一个 MCP Server 可以在任何支持 MCP 的 Host 中复用

对本系列读者来说，最直接的感受就是：skills 可以依赖 MCP server 获得文件操作、联网搜索、GitHub 管理等能力，这也是为什么高星 skills 几乎都依赖 MCP。

.mcp.json 配置详解

配置位置

MCP server 的配置写在 JSON 文件中，分两层：

层级 路径 提交 Git 用途
项目（共享） .claude/.mcp.json ✅ 团队共享的 MCP server 配置
全局（个人） ~/.claude/.mcp.json ❌ 个人全局 MCP server，所有项目生效

项目配置优先于全局配置。两层的 server 会自动合并——同名 server 以项目级为准。

.mcp.json 位于 .claude/ 目录下，与 CLAUDE.md、settings.json 平级（参见本系列 基础08：.claude目录）。

配置格式

mcpServers 是一个字典，key 是自定义的 server 名称（可随意命名），每个 server 的字段如下：

字段对照表

字段 必填 类型 说明
command ✅ string 启动 MCP server 的可执行命令（npx / node / python / uvx 等）
args 否 string[] 传给 command 的命令行参数数组
env 否 object 环境变量字典（适合存放 API Key 等敏感信息）
transport 否 string 传输方式："stdio"（默认）或 "sse"
disabled 否 boolean true 时禁用该 server（保留配置但不启动）
description 否 string 描述，便于其他人理解此 server 的用途

transport：stdio vs SSE

传输方式 说明 适用场景
stdio 父子进程通过标准输入/输出通信 本地开发首选，简单高效，无需网络
sse Server-Sent Events，通过 HTTP 通信 远程 Server、多 Host 共享、容器化部署

日常使用选 stdio 就对了。除了 transport 字段不同，两种方式的配置结构完全一致。

配置多个 server

args 中传递的是 server 的运行参数，各 server 含义不同。例如 Filesystem server 的末尾参数是允许操作的目录白名单。

安全提示

  • 不要在 .mcp.json 中硬编码 API Key / Token——用 env 字段配合环境变量引用，或 .mcp.json 提交到 Git 但 Token 通过 ~/.bashrc/.env 注入
  • Filesystem server 的目录白名单要给最小权限，只开放需要的目录

配置后如何验证

添加或修改 .mcp.json 后，有两种方式确认 MCP server 是否正常工作：

方式一：看启动日志

如果某个 server 连接失败，会有明确的错误信息（如 timeout、command not found、ENOENT）。

方式二：对话中查看可用工具

直接问 Claude：

你现在有哪些 MCP 工具可以用？

Claude 会列出所有已连接的 MCP server 及其提供的工具清单。如果某个 server 的工具没出现，说明它没启动成功。

常见启动失败原因：

  • command 拼写错误或路径不对
  • 缺少依赖（如未安装 npx 或对应的 npm 包）
  • args 格式错误（注意是数组，不是字符串）
  • env 中依赖的环境变量未设置

实例一：Filesystem 服务器

Filesystem 是 Anthropic 官方提供的 MCP server，能力包括：文件读写、搜索、目录遍历、批量操作等。它被大量 skills 依赖作为文件操作基础设施。

配置

  • "."：允许操作当前项目目录
  • "/tmp"：允许操作临时目录
  • 可以列多个目录白名单

如果 npx 拉取太慢，可以先全局安装：npm install -g @modelcontextprotocol/server-filesystem，然后 command 改为直接调用。

安装验证

配置好后，启动 Claude Code，在对话中可以看到多出了一个工具组。你可以直接提问：

帮我列出当前目录下所有 Markdown 文件，按大小排序

Claude 会在 Filesystem server 的支持下执行文件搜索和统计。

效果演示

文件搜索：

搜索 source/_posts/ 下所有包含 “MCP” 的文章

→ Claude 调用 Filesystem 的 search 工具，返回匹配结果。

内容读取：

把 package.json 的 scripts 部分读给我看

→ Claude 调用 read_file 工具。

批量操作：

把 img/ 目录下所有 .webp 文件的大小统计一下

→ Claude 遍历目录并汇总。

为什么 skills 依赖它

很多 skills 的核心能力就是操作文件——搜索代码、读取配置、写入结果。如果没有 Filesystem MCP Server，这些 skill 只能通过 Bash 调 shell 命令来完成（效率低、容易出错、跨平台不一致）。

Filesystem 提供了安全的沙箱文件操作接口，Claude 可以精确地读写白名单内的文件，无需直接操作 shell，更安全也更可靠。

实例三：GitHub 服务器

GitHub MCP server 让你可以通过自然语言管理仓库、Issue、PR 等。它被 code-review、handoff 等协作类 skills 依赖。

配置

需要先准备 GitHub Personal Access Token：

  1. 访问 GitHub → Settings → Developer settings → Personal access tokens → Fine-grained tokens
  2. 创建一个 token，权限选择你要操作的仓库
  3. 如果只需要公开仓库只读权限，选 public_repo 即可

GITHUB_TOKEN 跟 Brave Search 的 API Key 一样，建议通过环境变量注入，不要硬编码。

效果演示

配置好之后，你可以直接在 Claude Code 里操作 GitHub：

查看 Issue：

→ Claude 调用 GitHub server 列出 Issue。

管理 PR：

→ Claude 获取 PR diff 并进行分析。

仓库信息：

这个仓库的 star 数是多少？最近一次 commit 是什么？

→ Claude 直接查询 GitHub API 获取实时数据。

Skills 如何发现和利用 MCP Server

理解了这个机制，你就知道为什么高星 skills 几乎都依赖 MCP 了：

以 deep-research 为例，它的 SKILL.md 中会提示用户先配置 Brave Search 和 Fetch 两个 MCP server。配置之前，skill 会提示缺少依赖；配置之后，skill 自动获得联网搜索能力。

这就是 grill-me skill 为什么在你开始任务之前会先"采访"你——有些技能需要先根据你的回答，再结合 MCP 工具的能力来制定执行策略。

快速检查清单

配好一个 MCP server 后，确认以下几项：

检查项 方法
✅ Server 进程启动了 查看 Claude Code 启动日志
✅ 工具已注入 问 Claude “你现在有哪些 MCP 工具”
✅ 工具能正常调用 让 Claude 执行一个具体任务（如 “搜索某个关键词”）
✅ 配置文件安全 API Key 没硬编码在 .mcp.json 中
