ClaudeCode基础06：skills推荐

发表于2026-06-15|更新于2026-06-26|VibeCodingClaudeCode
|总字数:1.5k|阅读时长:5分钟|浏览量:

VERSION：3.0.0

caveman

链接：JuliusBrussee/caveman | ~20K+ stars

作用：让 Claude 说最少的话做最多的事，每次对话节省约 75% token。支持 lite/full/ultra/wenyan 多档强度。

用法：输入 /caveman 切换。之后 Claude 回复极度精简，只给代码和结论。/caveman ultra 更强，/caveman wenyan 文言文模式。再次输入 /caveman 恢复正常。

grill-me

链接：mattpocock/skills - grill-me

作用：写代码之前，先让 Claude 沿着决策树逐条追问你的设计细节，每个问题附带推荐答案，达成一致后再动手。

用法：开始新功能或改动前输入 /grill-me。Claude 逐个追问方案细节，回答完毕后自动进入实现阶段。

handoff

链接：mattpocock/skills - handoff

作用：把当前会话的决策记录、未完成任务打包成文档，让另一个会话无缝接续。自动脱敏 API Key 和密码。

用法：需要切换上下文时输入 /handoff。生成的文档路径输出到终端，新会话中可粘贴或通过 --continue 接续。

prototype

链接：mattpocock/skills - prototype

作用：不确定方案能不能跑通？先让 Claude 快速生成最小化原型验证核心逻辑，再投入正式开发。

用法：输入 /prototype "我想验证..."。Claude 快速生成可运行的原型，验证可行后再用正常聊天正式实现。

diagnosing-bugs

链接：mattpocock/skills - diagnosing-bugs

作用：遇到 bug 别急着改代码，系统性 5 步法找到根因再修，避免「改了一堆但问题不在这里」。

用法：遇到顽固 bug 时输入 /diagnosing-bugs。Claude 引导你走完复现 → 最小化 → 假设 → 验证 → 修复 + 回归测试的完整循环。

git-guardrails

链接：mattpocock/skills - git-guardrails-claude-code

作用：给 Claude 的 Git 操作装上护栏，执行 git push --force、git reset --hard 等危险操作前拦截确认。

用法：安装后自动生效，无需手动调用。Claude 执行危险 Git 命令时会弹确认，确认后才放行。

guard

链接：garrytan/gstack - guard

作用：全局安全网——拦截 rm -rf 等破坏性系统命令，额外提供 /freeze <目录> 锁定目录禁止 AI 修改。

用法：安装后自动生效。需要保护目录时输入 /freeze src/config/，被保护目录 Claude 只读不写。

office-hours

链接：garrytan/gstack - office-hours

作用：YC 掌门人 Garry Tan 的产品追问模板——6 个强制问题把你的模糊想法打磨成明确方向。

用法：开始新需求前输入 /office-hours。Claude 逐一抛出 6 个问题（为什么要做？用户是谁？成功标准？等），回答完再开工。

writing-great-skills

链接：mattpocock/skills - writing-great-skills

作用：让 Claude 引导你描述需求，按正确结构生成完整的 SKILL.md，学会后自己源源不断地创造定制 skill。

用法：输入 /writing-great-skills，用自然语言描述你想要的新技能，Claude 生成 SKILL.md 后立即生效。

improve-codebase-architecture

链接：mattpocock/skills - improve-codebase-architecture

作用：基于「浅模块 vs 深模块」理论，自动扫描代码库识别设计恶化点，生成带 Mermaid 对比图的 HTML 报告。

用法：定期输入 /improve-codebase-architecture。Claude 扫描后生成报告，可进一步 grill 讨论优化方案。

domain-modeling

链接：mattpocock/skills - domain-modeling | grill-with-docs

作用：Grill-Me 进阶版，盘问同时维护项目术语表和架构决策记录（ADR）。

用法：接手遗留项目或做重大架构变更时输入 /domain-modeling。盘问过程中自动更新 CONTEXT.md 和 docs/adr/。

claude-memory-skill

链接：hanfang/claude-memory-skill

作用：极简 Markdown 记忆系统——无数据库、无向量嵌入、零外部依赖，三层结构记忆核心知识、主题分类和用户偏好。

用法：安装后自动生效。会话启动时自动读取记忆文件，结束后将本次学到的偏好和知识追加写入。

andrej-karpathy-skills

链接：multica-ai/andrej-karpathy-skills

作用：一份 CLAUDE.md，四条原则（Think Before Coding / Simplicity First / Surgical Changes / Goal-Driven Execution），修复 LLM 编码的四个通病。

用法：下载到项目根目录或 ~/.claude/ 后自动生效。Claude 自动遵循四条原则。可编辑 CLAUDE.md 调整或移除规则。

SuperPowers

链接：obra/superpowers

作用：最成熟的 Claude Code 技能框架，14+ 个子技能覆盖 brainstorm、TDD、系统性调试等工作流。

用法：推荐流程：/brainstorm → /write-plan → /execute-plan → /code-review。也可单独使用 /systematic-debug 排查 bug 或 /tdd 做测试驱动开发。

子技能 用途
/brainstorm 苏格拉底式提问，把模糊需求敲定成清晰 spec
/write-plan 生成超详细的实现计划
/execute-plan 按计划逐批执行，每批有人工检查点
/tdd 严格 RED → GREEN → REFACTOR
/systematic-debug 4 阶段根因分析
/subagent-dev 每个任务分派独立 subagent
/parallel-agents 前端 + 后端 + 测试并行开发
/code-review 自动审查代码
/git-worktree 每个 feature 用独立 worktree 隔离

Semble

链接：MinishLab/semble

作用：AI Agent 专用的语义代码搜索引擎，用自然语言搜代码而非 grep + cat。全部本地 CPU 运行，单次查询约 1.5ms。

用法：安装后作为 MCP 工具自动生效。Claude 自动调用 search 和 find_related 工具。你也可以直接说"帮我找到用户认证相关的代码"。

CodeGraph

链接：colbymchenry/codegraph

作用：预构建代码知识图谱，把函数调用关系、符号引用、类型依赖存入 SQLite，Claude 可即时查询影响范围。

用法：安装后 Claude 自动使用 codegraph_impact（修改前查看影响范围）、codegraph_trace（追踪调用链）等工具。

Context-Mode

链接：mksglu/context-mode

作用：上下文窗口优化器，通过 sandbox 执行、知识库索引、session 断点续传把上下文消耗降低 98%。

用法：安装后在后台自动运行。可用 /context-mode:ctx-stats 查看 token 消耗明细，/context-mode:ctx-doctor 诊断运行状态。
