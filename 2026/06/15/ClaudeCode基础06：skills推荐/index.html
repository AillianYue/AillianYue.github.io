ClaudeCode基础06：skills推荐

发表于2026-06-15|更新于2026-06-15|VibeCodingClaudeCode
|总字数:5.4k|阅读时长:20分钟|浏览量:

VERSION：1.2.0

复制即用型

Grill-Me

SKILL 文件：mattpocock/skills - grill-me

一句话：写代码之前，先让 Claude 盘问你的方案，防止方向错了。

工作原理：Claude 不会直接开写，而是沿着决策树深度优先地逐个追问你的每个设计细节，每个问题附带推荐答案，双方达成一致后再动手。

SKILL.md：

使用方法：在开始新功能或改动前输入 /grill-me。Claude 会逐条追问你的方案细节，回答完毕后自动进入实现阶段。

Handoff

SKILL 文件：mattpocock/skills - handoff

一句话：把当前会话上下文打包成文档，让另一个 Claude 或自己下次无缝接续。

工作原理：压缩会话中的决策记录、未完成任务、关键上下文 → 生成 handoff 文档（存 OS 临时目录） → 自动脱敏 API Key 和密码，附带 suggested skills 建议。

SKILL.md：

使用方法：在需要切换上下文时输入 /handoff。生成的文档路径会输出到终端，新会话中可以将文档内容粘贴或通过 --continue 接续。

domain-modeling（Grill-With-Docs）

SKILL 文件：mattpocock/skills - grill-with-docs，mattpocock/skills - domain-modeling

一句话：Grill-Me 的进阶版，盘问的同时维护项目术语表和架构决策记录。

工作原理：在 grill 的基础上增加三项能力——创建/更新 CONTEXT.md 建立项目术语表，决策定型时内联写入 ADR（架构决策记录），对照现有代码库检验你的计划。

安装方式：

使用方法：接手遗留项目或做重大架构变更时输入 /domain-modeling。盘问过程中 CONTEXT.md 和 docs/adr/ 会被自动更新。

Improve-Codebase-Architecture

SKILL 文件：mattpocock/skills - improve-codebase-architecture

一句话：基于《软件设计哲学》的「浅模块 vs 深模块」理论，自动扫描代码库找出设计恶化点。

工作原理：扫描代码库识别出模块接口复杂度接近实现复杂度的浅模块 → 生成 HTML 报告（含 Mermaid 对比图） → 支持进一步 grill 讨论优化方案 → 可选生成 GitHub Issue。

安装方式：

使用方法：定期（建议每月或感觉代码库变混乱时）输入 /improve-codebase-architecture。Claude 会自动扫描并生成报告，审阅后选择需要优化的模块进行重构。

SuperPowers

GitHub：obra/superpowers

一句话：最成熟的 Claude Code 技能框架，14+ 个子技能覆盖 brainstorm、TDD、系统性调试等工作流。

核心子技能一览：

子技能 用途
/brainstorm 苏格拉底式提问，把模糊需求敲定成清晰 spec
/write-plan 生成超详细的实现计划（每个任务 2-5 分钟）
/execute-plan 按计划逐批执行，每批有人工检查点
/tdd 严格 RED → GREEN → REFACTOR，拒绝在测试之前写代码
/systematic-debug 4 阶段根因分析：收集证据 → 形成假设 → 实验 → 修复验证
/subagent-dev 每个任务分派独立的 subagent，隔离上下文
/parallel-agents 前端 + 后端 + 测试并行开发
/code-review 自动审查代码，Critical 级别阻塞合并
/git-worktree 每个 feature 用独立 worktree 隔离变更

安装方式：

使用方法：根据当前需要直接调用子技能。推荐流程：/brainstorm → /write-plan → /execute-plan → /code-review。也可以单独使用 /systematic-debug 排查 bug 或 /tdd 做测试驱动开发。

Taste-Skill

GitHub：Leonxlnx/taste-skill

一句话：给 AI 装审美，阻止 Claude 生成千篇一律的「AI 风格」UI。

工作原理：通过 SKILL.md 指令文件定义前端视觉规则，核心是三个「品味旋钮」（Taste Dials），分别控制布局实验性、动效强度和信息密度。

旋钮 范围 效果
DESIGN_VARIANCE 1-10 布局实验性（1=居中安全，10=不对称/激进）
MOTION_INTENSITY 1-10 动效强度（1=微妙，10=影院级物理动画）
VISUAL_DENSITY 1-10 信息密度（1=宽松/奢华，10=仪表盘/紧凑）

安装方式：

使用方法：在对话中告知 Claude 激活对应的 taste skill（如 “design-taste-frontend”）。Claude 会在生成前端代码时遵循预设的设计规则。可通过调整 SKILL.md 中的 DESIGN_VARIANCE / MOTION_INTENSITY / VISUAL_DENSITY 参数控制风格倾向。

diagnosing-bugs

SKILL 文件：mattpocock/skills - diagnosing-bugs

一句话：遇到 bug 别急着改代码，先按 5 步法系统性地找到根因再修。

工作原理：复现问题 → 最小化复现步骤 → 形成假设 → 插入检查点验证 → 修复 → 写回归测试。每一步 Claude 确认后再继续，避免「改了一堆但问题其实不在这里」的浪费。

安装方式：

使用方法：遇到不认识的 bug 时输入 /diagnosing-bugs。Claude 不会直接开修，而是引导你走完完整的诊断循环。适合「看着眼熟但怎么改都不对」的顽固 bug。

caveman

GitHub：JuliusBrussee/caveman | ~20K+ stars

一句话：让 Claude 说最少的话做最多的事，每次对话节省 ~75% token 开销。全网最火的单文件 skill 之一。

工作原理：强制 Claude 用极简语言输出——不解释、不铺垫、不总结，只说结论和代码。支持多档强度：lite（精简专业）、full（石器时代·默认）、ultra（极限缩写）、wenyan（文言文模式）。

SKILL.md（单个文件，直接复制到 .claude/skills/caveman/SKILL.md）：

安装方式：

使用方法：有经验的开发者想快速推进时输入 /caveman。之后 Claude 的回复极度精简，只给代码和关键结论。支持强度切换：/caveman lite、/caveman ultra，甚至 /caveman wenyan（文言文模式）。再输入一次 /caveman 切回正常模式。

git-guardrails-claude-code

SKILL 文件：mattpocock/skills - git-guardrails-claude-code

一句话：给 Claude 的 Git 操作装上护栏，防止误执行 destructive 命令。

工作原理：注册 PreToolUse hook，在执行 git push --force、git reset --hard、git clean -fd、git branch -D 等危险操作前拦截并让你确认。保底安全网。

安装方式：

使用方法：安装后自动生效，无需手动调用。当 Claude 打算执行危险 Git 命令时弹确认，确认后才放行。适合所有人——不怕一万就怕万一。

prototype

SKILL 文件：mattpocock/skills - prototype

一句话：不确定方案能不能跑通？先做个最小化原型验证，再投入正式开发。

工作原理：Claude 基于你的描述快速生成可运行的原型——要么是可执行的终端程序验证核心逻辑，要么是多个 UI 变体塞在一个路由下方便对比。原型不计入最终代码质量，但帮你确认方向没错。

安装方式：

使用方法：面对不熟悉的技术栈或不确定的架构方案时，输入 /prototype "我想验证..."。Claude 会快速生成原型，验证可行后再用正常聊天正式实现。适合「先看看能不能跑」的场景。

office-hours

SKILL 文件：garrytan/gstack - office-hours

一句话：YC 掌门人 Garry Tan 的产品追问模板——开始之前先被盘问 6 个问题，想不清楚不许动手。

工作原理：6 个强制问题——为什么要做？用户是谁？成功标准是什么？不做会怎样？最坏情况？如何验证？——把你的模糊想法打磨成明确方向。比 grill-me 更偏产品思维，适合非代码决策场景。

安装方式：

使用方法：开始新需求或新功能前输入 /office-hours。Claude 逐个抛出 6 个问题，你回答完后再开工。适合「大概知道要做什么但仔细一想又说不清」的情形。

writing-great-skills

SKILL 文件：mattpocock/skills - writing-great-skills

一句话：让 Claude 帮你写新的 SKILL.md，学会后你可以自己源源不断地创造定制 skill。

工作原理：Claude 引导你描述想要的新技能的功能和触发条件，然后按照正确结构（YAML frontmatter、渐进式指令、references/ 目录）生成完整的 SKILL.md 文件。写完后直接作为 slash command 使用。

安装方式：

使用方法：输入 /writing-great-skills 后用自然语言描述你想要的新技能（如「每次写 Python 时自动加上类型注解和 docstring」）。Claude 生成 SKILL.md 后立即生效，你就能当普通 slash command 调用了。

guard

SKILL 文件：garrytan/gstack - guard

一句话：全局安全网——阻止危险命令 + freeze 目录级写保护，让你放心让 Claude 放手干活。

工作原理：拦截 destructive 系统命令（rm -rf、强制推送到远端等），执行前要求你确认。额外提供 /freeze <目录> 和 /unfreeze 命令，锁定某些目录禁止 AI 修改（如数据库 schema、CI 配置）。

安装方式：

使用方法：安装后自动生效。需要保护某个目录时输入 /freeze src/config/，被保护目录 Claude 只读不写。配合 git-guardrails 一起用效果更佳。

claude-memory-skill

GitHub：hanfang/claude-memory-skill

一句话：极简 Markdown 记忆系统——无数据库、无向量嵌入、零外部依赖，curl 一行命令装好。

工作原理：在 .claude/skills/claude-memory/ 下维护三层结构——core.md 存核心知识、topics/*.md 按主题分类、me.md 记用户偏好。会话启动时注入记忆，结束时自动提取新心得追加写入。

安装方式：

使用方法：安装后自动生效。每次会话开始时 Claude 自动读取记忆文件，结束后将本次学到的偏好和知识追加写入。你也可以随时查看 ~/.claude/skills/claude-memory/ 下的文件手动编辑。

MCP Server 型

这组 skill 需要安装对应的运行时（npm 或 pip）并注册 MCP server 到 Claude Code。安装后 Claude 会获得额外的工具能力，在对话中自动调用。

Semble

GitHub：MinishLab/semble | 官网：https://minish.ai/packages/semble/introduction/

一句话：AI Agent 专用的语义代码搜索引擎，用自然语言搜代码而非 grep + cat。

工作原理：tree-sitter 按函数/类粒度切块 → Model2Vec 静态嵌入 + BM25 混合检索 → code-aware reranking。索引一个中型仓库约 250ms，单次查询约 1.5ms，全部本地 CPU 运行。

安装方式：

使用方法：Semble 安装后自动作为 MCP 工具生效，无需手动调用。当 Claude 需要查找代码时，它会自动使用 Semble 的 search 和 find_related 工具进行语义搜索。你也可以在对话中用自然语言描述需求，如"帮我找到用户认证相关的代码"，Claude 会自动调用 Semble 完成搜索。

CodeGraph

GitHub：colbymchenry/codegraph

一句话：预构建代码知识图谱，把函数调用关系、符号引用、类型依赖存储到 SQLite 中供 Claude 即时查询。

安装方式：

使用方法：安装后 Claude 会自动使用 codegraph_context、codegraph_trace、codegraph_impact 等工具查询代码。常见用法：

  • 修改函数前自动查看 codegraph_impact 分析影响范围
  • 理解复杂调用链时自动使用 codegraph_trace 追踪 “A 如何到达 B”
  • 初次安装后建议运行 codegraph 构建索引，后续文件变更会自动增量同步

Context-Mode

GitHub：mksglu/context-mode

一句话：上下文窗口优化器，通过 sandbox 执行工具输出、知识库索引、session 断点续传把上下文消耗降低 98%。

工作原理：注册 PreToolUse / PostToolUse / PreCompact / SessionStart 等 hooks，在后台拦截工具输出压缩后存入上下文。会话压缩时生成 ≤2KB 的 XML Session Guide，下次 --continue 时注入恢复上下文。

安装方式：

使用方法：安装后自动在后台运行，无需手动干预。可通过以下命令查看状态：

  • /context-mode:ctx-stats — 查看每个 tool 的 token 消耗明细
  • /context-mode:ctx-doctor — 诊断 hooks 和运行时是否正常工作
  • ctx_execute 和 ctx_batch_execute 是内置的 sandbox 执行工具，Claude 会在需要运行代码时自动调用它们以减少上下文消耗

环境配置型

这组 skill 不依赖 .claude/skills/ 目录，而是通过修改项目文件或目录结构来生效。

OpenSpec

GitHub：Fission-AI/OpenSpec | 官网：https://openspec.dev

一句话：规范驱动开发（SDD）框架，先写 spec 再写代码最后归档。

三阶段工作流：

阶段 命令 产出
Propose /openspec:proposal "..." proposal.md + spec + design.md + tasks.md
Apply /openspec:apply <change-name> 按 tasks.md 逐条执行，每步跑测试
Archive /openspec:archive <change-name> 归档变更，合并 spec 到知识库

安装方式：

使用方法：按三个阶段依次执行：

  1. /openspec:proposal "你的变更描述" — 生成 spec 和任务清单，审阅后进入下一步
  2. /openspec:apply <change-name> — Claude 按 tasks.md 逐条执行，每完成一步自动跑测试
  3. /openspec:archive <change-name> — 归档变更，spec 合并到项目知识库

andrej-karpathy-skills

GitHub：multica-ai/andrej-karpathy-skills

一句话：一份 CLAUDE.md 文件，四条原则，修复 LLM 编码的四个通病。

原则 解决什么问题
Think Before Coding 隐藏假设、未言明的取舍
Simplicity First 过度工程、不必要的抽象
Surgical Changes 改一个功能顺手改了三处无关代码
Goal-Driven Execution 写了一大堆但没验证对不对

安装方式：

使用方法：安装后自动生效，无需手动调用。Claude 在每次对话中会自动遵循 Think Before Coding / Simplicity First / Surgical Changes / Goal-Driven Execution 四条原则。如果某些原则不符合你的项目风格，可以直接编辑 CLAUDE.md 调整或移除对应规则。
