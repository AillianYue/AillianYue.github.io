ClaudeCode基础09：hooks

发表于2026-06-18|更新于2026-06-26|VibeCodingClaudeCode
|总字数:3.2k|阅读时长:13分钟|浏览量:

VERSION：1.0.1

hooks 是什么

hooks 是 Claude Code 的事件系统——在特定时机（工具执行前/后、会话启动、收到通知等）强制自动执行你指定的脚本或命令。

简单说：hooks = 挂钩事件 + 自动执行脚本。

它跟 CLAUDE.md 的区别在于：

机制 本质 由谁执行
CLAUDE.md 给 AI 读的指令（建议性） AI 自觉遵守
hooks 系统级触发（强制性） 你的 Shell，不由 AI 控制

hooks 在 LLM 之外运行，所以更可靠：LLM 可能"忘记"规则，但 hooks 每次都会触发。

配置位置

hooks 写在 JSON 配置中，分三层（优先级从高到低）：

层级 路径 提交 Git 用途
本地（个人） .claude/settings.local.json ❌ 个人偏好、本地环境差异
项目（共享） .claude/settings.json ✅ 团队统一规则
全局（个人） ~/.claude/settings.json ❌ 个人全局偏好，所有项目生效

同名配置项合并：数组相加、标量本地 > 项目 > 全局。

配置格式

Event

事件名即触发时机，目前的 27 个事件可分为几大类：

类别 事件 触发时机
会话 SessionStart 会话启动或恢复时
SessionEnd 会话结束时
工具 PreToolUse 任何工具调用之前（最常用的安全拦截点）
PostToolUse 任何工具调用之后（自动格式化、日志等）
PostToolUseFailure 工具调用失败时
提示词 UserPromptSubmit 用户提交提示词后、AI 处理前
通知 Notification Claude 需要用户注意时（等待输入等）
停止 Stop AI 完成一轮回答时
StopFailure AI 回答异常结束时
压缩 PreCompact 上下文即将压缩前
PostCompact 上下文压缩完成后
子代理 SubagentStart 子代理启动时
SubagentStop 子代理完成时

顶层字段

每个事件下是一个数组，数组元素是两个字段：

字段 必填 说明
matcher 否 工具匹配规则，为空或 "*" 则匹配全部（见下方详解）
hooks ✅ 要执行的 hook 数组（可绑定多个）

matcher（匹配器）

控制 hook 在哪些工具调用时才触发。支持三种写法：

写法 示例 说明
工具名精确匹配 "Bash" 仅匹配 Bash 工具
工具名多选 "Edit|Write" 匹配 Edit 或 Write 工具
正则表达式 "^Notebook.*" 匹配所有 Notebook 开头的工具

注意：UserPromptSubmit、Stop、SessionStart 等非工具事件上的 matcher 会被忽略，写了也白写。

hooks 数组元素字段

每个事件可挂多个 hook，每个 hook 的字段如下：

字段 必填 类型 说明
type ✅ string 处理类型：command、http、mcp_tool、prompt、agent
command 条件必填 string shell 命令（type: command 时必填），通过 stdin 接收 JSON 上下文
async 否 boolean 是否异步运行不阻塞主流程（仅 type: command 有效）
timeout 否 number 超时秒数。默认值：command=600, http=30, prompt=30, agent=60
statusMessage 否 string hook 运行时显示的自定义加载文字
once 否 boolean 是否每个会话只运行一次（仅对 skill/agent 前注有效，settings.json 中无效）
if 否 string 权限规则表达式过滤（仅限工具事件）
shell 否 string "bash"（默认）或 "powershell"（Windows）

type 的 5 种取值

类型 说明 支持 async 默认超时
command 执行 shell 脚本，通过 stdin 接收 JSON 上下文 （ 最常用） ✅ 600s
http POST 请求到指定 URL ❌ 30s
mcp_tool 调用已注册的 MCP 工具 ❌ —
prompt 单轮 LLM 评估（让 AI 判断是否放行） ❌ 30s
agent 多轮子代理（有工具使用权限） ❌ 60s

新手阶段，用 command 类型就够了，所有实战例子都是它。

hook 脚本如何与 Claude 通信

command 脚本通过 stdin 接收 JSON、通过 退出码和 stdout 控制行为：

退出码含义

退出码 含义
0 允许继续（可以输出 JSON 修改工具入参）
1 脚本错误（会阻止操作）
2 阻止操作**（仅对 PreToolUse 有效，stderr 提示会反馈给 AI）
其他非 0 非阻塞错误（显示提示但不阻止）

输出 JSON 控制返回

stdout 输出 JSON 可精细控制行为：

字段 说明
decision "allow" 放行 / "block" 阻止
reason 阻止时显示给用户的理由
modified_input 修改后的工具入参（decision 为 "allow" 时生效）
suppressOutput 是否隐藏本 hook 的输出
continue false 跳过后续该事件的 hook
feedback 非阻塞的提示信息

使用建议

从小开始

不要一次性上十几个 hook。推荐起步组合：

用途 Hook
安全 PreToolUse 防危险命令
自动化 PostToolUse 自动格式化
体验 SessionStart 启动问候
规范 UserPromptSubmit 注入上下文

保持快速

  • SessionStart hook 每次启动都跑——脚本不要超过 200ms。
  • 耗时操作用 async: true 异步运行（当前仅 type: command 支持）。
  • 所有 hook 默认最多 600s 超时，建议根据脚本复杂度设小一点的 timeout。

安全要点

  • hook 脚本以你的用户权限运行——写代码时假设可能被利用。
  • shell 变量记得加引号防止注入。
  • 用绝对路径引用关键脚本。
  • PreToolUse 的退出码 2 是最可靠的阻止方式。
